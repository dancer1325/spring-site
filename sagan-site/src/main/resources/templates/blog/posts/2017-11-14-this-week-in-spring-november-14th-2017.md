# This Week in Spring - November 14th 2017

_Engineering | Josh Long |  November 14, 2017 | 0 Comments_

Hi Spring fans! Welcome to another installment of *This Week in Spring*! This week I am in Casablanca, Morocco for the amazing Devoxx Morocco event talking to developers in the hub of North Africa.

Can you believe it's already the middle of November? And can you believe we are less than a month away from the [amazing SpringOne Platform](https://springoneplatform.io/) event in San Francisco, CA? I hope you're going!

Now, without further ado, let's get to it.

-   Spring messaging ninja Oleg Zhurakousky just announced [Spring Cloud Stream Elmhurst.M3 / 2.0.0.M3](https://spring.io/blog/2017/11/09/spring-cloud-stream-elmhurst-m3-2-0-0-m3-release-announcement). The new release improves content-type resolution, support for Apache Kafka 0.11+, and introduces a good deal of nice samples.
-   Spring Data Geode/GemFire lead John Blum [just announced Spring Data Geode / GemFire 2.0.0.RC1](https://spring.io/blog/2017/11/07/spring-session-data-geode-gemfire-2-0-0-rc1-available). The new release updates dependencies, builds on Java 8 (introducing Java 8 types and lambdas), and introduces new samples.
-   In this last week's installment of *Spring Tips* [I revisit Kotlin and Spring](https://spring.io/blog/2017/11/08/spring-tips-bootiful-kotlin-redux), looking at some of the DSLs available in Spring and their use. This one's been a long time in coming and it's wonderful to see how far things have come!
-   Spring messaging ninja Artem Bilan [just announced Spring for Apache Kafka 2.1RC1 and 2.0.1 and 1.3.1](https://spring.io/blog/2017/11/09/spring-for-apache-kafka-2-1-rc1-and-2-0-1-1-3-1-available) which - mainly - introduces support for Apache Kafka 1.0.
-   Spring Cloud Task contributor [Glenn Renfro just announced Spring Cloud Task 2.0.0.M2](https://spring.io/blog/2017/11/10/spring-cloud-task-2-0-0-m2-is-now-available). This release updates the code to be compatible with Spring Boot 2.0.0.M5 (and all the other downstream dependencies that depend on it).
-   This Devoxx BE talk on [consumer driven contract testing with Spring Cloud Contract](https://www.youtube.com/watch?v=_DkvVtDPiFM&list=PLRsbF2sD7JVqZ4RpHYkqSuCNhxumGP5eo&index=109) by Ben Ooms looks interesting
-   I happen to know that this talk by Keycloak contributor Sebastien Blanc [on using Spring Boot and RedHat's Keycloak](https://www.youtube.com/watch?v=3I4TXPxCCVE&list=PLRsbF2sD7JVqZ4RpHYkqSuCNhxumGP5eo&index=81) OAuth identity provider is worth a watch.
-   Spring Framework and Spring co-founder Juergen Hoeller gave a great talk at Devoxx Belgium [introducing Spring Framework 5 themes and trends](https://www.youtube.com/watch?v=69tJKcmKVQk&list=PLRsbF2sD7JVqZ4RpHYkqSuCNhxumGP5eo&index=72).
-   I also had fun co-presenting with my buddy Mark Heckler [on building reactive Spring applications at Devoxx BE](https://www.youtube.com/watch?v=yAXgkSlrmBA&list=PLRsbF2sD7JVqZ4RpHYkqSuCNhxumGP5eo&index=64).
-   This talk by Spring team legend Thomas Risberg [on how to build Spring Boot applications on Kubernetes](https://www.youtube.com/watch?v=EerHub6PoEA&list=PLRsbF2sD7JVqZ4RpHYkqSuCNhxumGP5eo&index=60) is defnitely worth watching.
-   I had a lot of fun presenting at Devoxx BE with my buddy and Okta Developer Advocate Matt Raible in this talk called [Building Cloud Native Progressive Web Apps](https://www.youtube.com/watch?v=tCyLDzhz_mg&list=PLRsbF2sD7JVqZ4RpHYkqSuCNhxumGP5eo&index=51). In it we look at using Spring Boot, Spring Cloud and Kotlin to build a robust microservices-based backend and then we build a TypeScript-based Angular client to talk to that microservices system.
-   This looks interesting - it's a Devoxx BE talk by Arne Vandamme and Steven Gentens on building web applications using something called [the Across Platform (with Spring)](https://www.youtube.com/watch?v=00Jn3d12L2M&list=PLRsbF2sD7JVqZ4RpHYkqSuCNhxumGP5eo&index=40). I've got it bookmarked!
-   You should definitely check out this talk on [Spring Framework 5 hidden gems](https://www.youtube.com/watch?v=LQN2ptVdaJE&list=PLRsbF2sD7JVqZ4RpHYkqSuCNhxumGP5eo&index=38) by Spring Framework lead Juergen Hoeller
-   I *love* this Devoxx BE talk by Kotlin and Spring ninja Sébastien Deleuze on, well, [Spring Boot and Kotlin](https://www.youtube.com/watch?v=kbMXAjWEft0&list=PLRsbF2sD7JVqZ4RpHYkqSuCNhxumGP5eo&index=36).
-   This Devoxx BE talk by Spring teammembers Stephane Nicoll and Brian Clozel on [building Spring Boot 2.0 web applications](https://www.youtube.com/watch?v=Q-9bUsrLVpI&list=PLRsbF2sD7JVqZ4RpHYkqSuCNhxumGP5eo&index=5) is a must-watch!
-   I really liked this Devoxx BE talk by Jisha Abubaker on Google's developer relations engineering team. In it she looks at [using some of the Google Cloud Spring Boot starters](https://www.youtube.com/watch?v=-pMxdcfWrhs&list=PLRsbF2sD7JVqZ4RpHYkqSuCNhxumGP5eo&index=22).
-   This post is interesting - Hibernate Developer Advocate Vlad Mihalcea looks at how to [simulate Spring Data's `@CreatedBy` and `@LastModifiedBy` with straight Hibernate](https://vladmihalcea.com/2017/11/14/how-to-emulate-createdby-and-lastmodifiedby-from-spring-data-using-the-generatortype-hibernate-annotation/amp/). Interesting.. but why not just use Hibernate *and* Spring Data JPA? N'er a more natural pair was there :)
-   This has zero things to do with Spring, but I'm honored to have been included [in this a list of twenty Java developers you should follow on Twitter](https://www.frgconsulting.com/blog/20-java-professionals-need-follow-twitter/). If you want to [follow l'il 'ol me](http://twitter.com/starbuxman), I'd appreciate it, but you should *definitely* follow some of the rest of the folks on that list. Most of them are friends and I can't recommend 'em highly enough.
