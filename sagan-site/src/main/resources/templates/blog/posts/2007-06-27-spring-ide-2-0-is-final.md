# Spring IDE 2.0 is Final

_Releases | Christian Dupuis |  June 27, 2007 | 0 Comments_

After fixing approximately 250 bugs and working uncountable hours on adding support for Spring 2.0, Spring Web Flow, Spring AOP and Spring JavaConfig, we are proud to announce the immediate availability of Spring IDE 2.0.  

![Spring IDE 2.0 Logo](http://springide.org/blog/wp-content/uploads/2007/06/springide_logo.png "Spring IDE 2.0 Logo")

[Download](http://springide.org/updatesite) | [Documentation](http://springide.org/project/wiki) | [Changelog](http://springide.org/project/wiki/ChangeLog)

The release is available from our [release update site](http://springide.org/updatesite). Spring IDE 2.0 is licensed under the terms of the [Eclipse Public License - v1.0](http://www.eclipse.org/legal/epl-v10.html).

**New Features**

Spring IDE 2.0 contains lots of new features and a bunch of bug fixes. A list of all closed tickets is available in our [ticketing system](http://springide.org/project/report/25). For those of you that are not familiar with recent development of Spring IDE here is a short list of features included:

-   Support for *Spring 2.0* namespace-based configurations. We have put lots of work into that to make the support as extensible as possible. You can read more about that in another [post](http://springide.org/blog/2007/04/05/adding-support-for-custom-namespaces/).
-   Support for *Spring Web Flow*, including an extension to WTP’s XML editor for content assist and hyperlinking as well as validation and graphical editing. More information is available [here](http://springide.org/blog/2007/03/13/spring-ide-m3-web-flow-support-has-arrived/).
-   [Tools](http://springide.org/blog/2007/01/27/insights-of-spring-ides-aop-support/) for *Spring AOP* based development. This includes support for validating configurations (parsing of pointcut expressions) and visualization of cross cutting references based on <aop:config> and @AspectJ-style aspects.
-   Support for *Spring JavaConfig* M2. This serves as sandbox for testing the extension points of Spring IDE’s core. Read more about that [here](http://springide.org/blog/2007/05/18/support-for-spring-javaconfig/) and [here](http://blog.interface21.com/main/2007/06/05/more-on-java-configuration/).
-   [Usability](http://springide.org/blog/2007/04/13/spring-ide-20-brings-improved-project-explorer-integration/) and UI improvements: A new *Spring Explorer* that replaces the [Beans View](http://springide.org/project/wiki/BeansView), Content contribution to the Eclipse’s Project Explorer, a *Spring Working Set* type to reduce cluttering in the Project and [Spring Explorer](http://springide.org/project/wiki/SpringExplorer), *Refactoring participants* for rename and move refactorings of Java Packages and Classes as well as Bean names, New Project and Spring Bean configuration file wizard.

Spring IDE 2.0 is compatible with upcoming Eclipse 3.3 (aka Eclipse Europa).

Read more at the [Spring IDE Blog](http://springide.org/blog/2007/06/27/spring-ide-20-is-final/).
