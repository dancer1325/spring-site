# dm Server 2.0 M2

_Engineering | Andy Wilkinson |  May 19, 2009 | 0 Comments_

[dm Server](http://www.springsource.org/dmserver) 2.0.0.M2 has been released, and is now available for [download](http://www.springsource.com/download/community?project=SpringSource%20dm%20Server&version=2.0.0.M2). In the two sprints since 2.0.0.M1, we've made significant progress towards the final release, both in terms of new features, and in defect fixes. Take a look at the [M2 release notes](https://issuetracker.springsource.com/secure/RunPortlet.jspa?portletKey=com.pyxis.greenhopper.jira:greenhopper-releasenotes&projectid=10054&fields=,summary&versionId=10172&includeSubs=false) if you're interested in seeing everything that we've been working on. Please keep your feedback coming as comments on our blog, in the [forums](http://forum.springsource.org/forumdisplay.php?f=53), and on [JIRA](https://issuetracker.springsource.com/browse/DMS).

## New and noteworthy

### Spring 3 support

In the milestone, we've upgraded dm Server to run on Spring 3. As we move towards the final release of dm Server 2.0, we'll be upgrading the version of Spring 3 as new milestones and snapshots become available.

### Improvements to cloning

We've made some significant improvements in our support for auto-cloning. For example, you can now run Spring 2.5.6 and Spring 3-based applications side-by-side in the same dm Server.

### Watched repositories

Support for watched repositories has been added. A watched repository is configured to watch a directory on your filesystem and, as artifacts are added to the directory, they will become available to dm Server's provisioning system. Equally, as artifacts are removed from the directory, they will no longer be available to dm Server's provisioning system.

Here's some example configuration for a watched repository:

```js
Copy"watched-repo" : {
    "type" : "watched",
    "watchDirectory" : "repository/watched",
    "watchInterval" : 5
}
```
