# This Week in Spring - Aug 6th, 2013

_Engineering | Josh Long |  August 07, 2013 | 0 Comments_

Welcome back to another installment of *This Week in Spring*. On August 1, I celebrated my third year at [SpringSource](http://twitter.com/Starbuxman). I continue to enjoy the ride of my life and a huge part of that is my interaction with you, the most amazing community ever. Thanks for that, folks.

Have you booked your tickets for **[SpringOne2GX](http://www.springone2gx.com/conference/santa_clara/2013/09/home)**? This year's show is a special one. In my work as the Spring Developer Advocate, I speak at *many* conferences all around the world. Ask any developer with a pulse, and they'll confirm that big data (and Hadoop), reactive web applications, REST, mobile application development, and cloud computing are sizzling hot topics today. Pivotal, and Spring, support today's developers, and SpringOne2GX's agenda represents in my estimation the perfect blend of content for today's developers. Check out [the agenda](https://springone2gx.com/conference/santa_clara/2013/09/springone/event_schedule). We've just recently added talks on [big data](http://www.springone2gx.com/conference/santa_clara/2013/09/session?id=29165), and [REST service security with OAuth](http://www.springone2gx.com/conference/santa_clara/2013/09/session?id=29425). This will be our first show under [Pivotal](http://gopivotal.com), and it's the only place where you can talk to the developers working on the technologies you care about both at SpringSource and in the community. As you know, we've just announced our Cloud Foundry conference, [Platform](http://platformcf.com), and SpringOne2GX full pass ticket holders may register for that show - which is at the same venue as SpringOne2GX just two days earlier - for free! If I had to pay for just one show a year, this would be that show. Hurry the early bird rate expires this friday!

1.  **Major news**: Phil Webb and Dr. David Syer have [just announced Spring Boot](http://blog.springsource.org/2013/08/06/spring-boot-simplifying-spring-for-everyone/), which simplifies Spring application development. Spring Boot provides an opinionated layer on top of Spring and in so doing makes it dead simple to get an application up and running with a minimum of fuss. Seriously, this stuff will blow your mind. Do not read any further until you've read this short and sweet post! Give it a go and be sure to [let us know about your experiences](http://twitter.com/SpringSource)!
2.  Spring Framework 3.2.4 maintenance release [is now available](http://www.springsource.org/node/22666), with an important security fix for SpringOXM..
3.  Spring Data Redis-lead and ninja Jennifer Hickey just announced the availability [of two Spring Data releases](http://www.springsource.org/node/22665). Spring Data Redis 1.1, M2, featuring a *lot* of new features, including enhanced data pipelining, Redis 2.6 scripting, and more. Spring Data Redis 1.0.6 is also available, and features bug fixes and smaller improvements.
4.  Spring Mobile and Android lead Roy Clarkson just announced [Spring Mobile 1.1.0.RC1](http://www.springsource.org/spring-mobile/news/1.1.0.rc1-released), which features improvements to device detection and view resolution in Spring Mobile. Roy also announced a new cut of the stable line of Spring Mobile, 1.0.2, which features similar improvements, some backported.
5.  Spring Data ninja Oliver Gierke has just announced that the final release candidate for [Spring Data Babbage](http://www.springsource.org/node/22662) is now available. This release is named for [Charles Babbage](http://en.wikipedia.org/wiki/Charles_Babbage). This release features support for the MongoDB Aggregation Framework and improved the execution of polymorphic queries, support to use SpEL expressions in manually defined queries with JPA, improved handling of entities using @IdClass, a `countBy(..)` method for Neo4j repositories, and *much* more.
6.  The replay for the webinars [*Functional Programming without Lamba*](http://www.springsource.org/node/22660) and [*Spring with Cucumber for Automation*](http://www.springsource.org/node/22661) are now available online. Be sure to check them out!
7.  A few weeks ago, our friend Johnathan Mark Smith put together a video introducing how to use [Spring Data MongoDB and Java configuration](https://www.youtube.com/watch?v=aY_JsfJM2eU&feature=share&list=UULXuFBD0Ik8VH10azZ--6Jg). Check it out! And, if you're doing awesome videos, feel free to share. I'd love to post them on *This Week in Spring*, too!
8.  I smiled when I saw a [tweet](https://twitter.com/j_brisbin/status/364774255826698243) by the [Reactor](http://github.com/reactor) project lead Jonathan Brisbin in which he says, "Processor throughput: 90M ops/sec on a laptop. 1 thread + @LMAX Disruptor. Not #fastdata, #uberfastdata" and then links to a [test case in the code](https://github.com/reactor/reactor/blob/master/reactor-core/src/test/java/reactor/core/processor/ProcessorThroughputTests.java#L71). Needless to say, [Reactor](http://blog.springsource.org/2013/05/13/reactor-a-foundation-for-asynchronous-applications-on-the-jvm/) is going to shake things up big time! (And, of course, we'll have more content on Reactor at [SpringOne2GX](http://springone2gx.com).
9.  The latest release of Tomcat, [Apache Tomcat 8.0.0-RC1 (alpha)](HTTP://ow.ly/nGwEo), is now available! There are a lot of new features. Notably, Tomcat 8 will be the first Tomcat to support JSR 356, web sockets. This is the perfect compliment to Spring 4's [recently announced web socket support](http://blog.springsource.org/2013/07/24/spring-framework-4-0-m2-websocket-messaging-architectures/).
10.  Mohan Srihari Kantipudi has put together a nice post on [Spring's basic REST capabilities](http://mohansrihari.wordpress.com/2013/08/05/ajax-request-processing-using-spring-mvc-3-x/)
11.  I liked Gregor Riegler's post on [Spring Loaded](http://babdev.blogspot.com/2013/08/good-bye-redeployment-spring-loaded.html), the best kept secret in open source. Spring Loaded is a Java agent that lets you reload code as you're working on it (no need to redeploy!). This is a *very* cool post and I hope you'll consider using Spring Loaded, too.
