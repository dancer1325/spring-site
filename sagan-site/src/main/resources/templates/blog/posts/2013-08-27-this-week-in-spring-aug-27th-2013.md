# This Week in Spring - Aug 27th, 2013

_Engineering | Josh Long |  August 27, 2013 | 0 Comments_

Welcome back to another installment of *This Week in Spring*. We have a lot to cover, as usual! Spring Security and lead Rob Winch feature heavily in this week's roundup! [So, cheers to Rob Winch](http://twitter.com/rob_winch)!

1.  Spring Security lead Rob Winch put together a [post](http://blog.springsource.org/2013/08/21/spring-security-3-2-0-rc1-highlights-csrf-protection/) introducing some of the new, smart and convenient protection in Spring Security against [cross-site request forgery (or *CSRF*)](http://wikipedia.org/wiki/csrf).
2.  Rob also put together *another* epic blog post that demonstrates some of Spring Security's [new support for security headers](http://blog.springsource.org/2013/08/23/spring-security-3-2-0-rc1-highlights-security-headers/).
3.  Rob was *also* kind enough to integrate these new features [into the Spring REST stack codebase](http://github.com/joshlong/the-spring-rest-stack) where you can see them in action in the context of a full-stack, integrated Spring REST service. To see these changes, along with Spring Security and Spring Security OAuth, all integrated using Java Configuration, check out the [`oauth` module](https://github.com/joshlong/the-spring-rest-stack/tree/master/code/oauth).
4.  Reactor lead Jonathan Brisbin has announced [Reactor 1.0.0.M2 is now available](http://blog.springsource.org/2013/08/27/reactor-1-0-0-m2-a-foundation-for-reactive-fast-data-applications-on-the-jvm/). The new release looks *very* exciting! From Jon's writeup: "This 2nd milestone includes a number of bugfixes and some really exciting new features. Reactor now includes a Processor abstraction, which is a highly-optimized task processor based on the [LMAX Disruptor RingBuffer](https://github.com/LMAX-Exchange/disruptor)… Anecdotal benchmarks on a MacBook Pro show the Processor can pump around 100,000,000 events per second through the pipeline. Yes, you read that right: **100 million per second**!"
5.  [Hyperic, Cloud Foundry, Spring and Spring Data ninja Jennifer Hickey](http://www.twitter.com/jencompgeek) has announced the [latest cut of the Spring Data Redis project](http://www.springsource.org/node/22678). The new release includes support for millisecond precision in key expiration commands, resubscription of message listeners on connection failure, a full implementation of `ConcurrentMap` contract in `RedisMap` and `RedisProperties`  
    
6.  Spring Batch lead [Michael Minella](http://twitter.com/michaelminella) has announced [Spring Batch 3.0M1 has been released](http://blog.springsource.org/2013/08/23/spring-batch-3-0-milestone-1-released/)! This release marks the first steps towards implementing the [JSR-352 Java Batch specification](http://jcp.org/en/jsr/detail?id=352), among other things.
7.  Register now for the Aug 29th Webinar: [Taming Coupling & Cohesive Beasts with Modularity Patterns and Spring](https://www.springsource.org/node/22645) with Param Rengaiah.
8.  Join our friends from Pivotal Labs as David Frank shows you How to [Get Agile with Pivotal Tracker](https://www.springsource.org/node/22670), on September 5th.
9.  Spring Security lead Rob Winch [tweeted](https://twitter.com/rob_winch/status/372484048603475968), "#Gradle made it dead simple to build #SpringSecurity with #SpringFramework 3 and run tests with both #Spring 3/4," and linked to [this epic example](https://github.com/SpringSource/spring-security/commit/aca2e4ff3a79a228ecf95fb36c4d255b72c39aca). This isn't strictly speaking Spring related post, but it's a nice example of a really elegant Gradle build, for those who also work with it, as we do at SpringSource.
10.  [Spring Security lead Rob Winch](http://twitter.com/rob_winch) (boy, that guy sure gets around!) also announced the latest [release of Spring Security LDAP](http://www.springsource.org/node/22680).
11.  Our pal Xavier Padró is back, this time with a post introducing how to use [resource-local JMS transactions on message receipt with Spring's `JmsTemplate`](http://xpadro.blogspot.com.es/2013/08/spring-jms-processing-messages-within.html).
12.  Patrick Grimard's put together a nice post on integrating [Yeoman Backbone with a Spring web application](http://jpgmr.wordpress.com/2013/08/26/yeoman-backbone-apps-meet-tuckeys-urlrewritefilter/), as well as a way to sidestep some issues he was having with resource resolution by using Tuckey’s `UrlRewriteFilter`. Now, I like the `UrlRewriteFilter`. That said, I think (but haven't tested..) that Patrick could've accomplished the same thing using Spring MVC resource handling support, as he starts to do in an example in the code when overriding the `public void addResourceHandlers(ResourceHandlerRegistry registry)` configuration method. Perhaps I'm mistaken, and either way, cool post!
13.  SpringSource has released new trainings to the Q3 schedule, check out the training schedules for: [Core Spring](http://mylearn.vmware.com/portals/www/search/results.cfm?ui=www_edu&menu=search-results&searchtype=simple&category=schedule&id_subject=17750), [Enterprise Integration with Spring](http://mylearn.vmware.com/portals/www/search/results.cfm?ui=www_edu&pID=www&menu=search-results&searchtype=simple&orderBy=relevance&category=schedule&keyword=enterprise+integration&Search=Search) and [Spring Web](http://mylearn.vmware.com/portals/www/search/results.cfm?ui=www_edu&pID=www&menu=search-results&searchtype=simple&orderBy=relevance&category=schedule&keyword=spring+web&Search=Search).
