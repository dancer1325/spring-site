# This Week in Spring - March 21st, 2017

_Engineering | Josh Long |  March 22, 2017 | 0 Comments_

Welcome to another installment of *This Week in Spring*! This week I'm visiting with customers and am presenting at the [first Devoxx US event in San Jose, California](http://ridingthecrest.com/blog/2017/03/19/devoxx-us-2017-picks.html). If you're around, don't hesitate to stop by and say hi at the Pivotal booth on the expo hall.

As usual, we've got a lot to cover so let's get to it!

-   [Spring Cloud Dalston RC1](https://spring.io/blog/2017/03/21/spring-cloud-dalston-rc1-released) is now available! Get the bits while they're hot!
-   Spring Cloud Task contributor [Glenn Renfro just announced Spring Cloud Task 1.2.0.M2](https://spring.io/blog/2017/03/21/spring-cloud-task-1-2-0-m2-is-now-available)
-   Marius Bogoevici just [announced Spring Cloud Stream Chelsea RC1](https://spring.io/blog/2017/03/16/spring-cloud-stream-chelsea-rc1-released)
-   Spring Cloud Vault [lead Mark Paluch just announced the 1.0.RC1 of the project](https://spring.io/blog/2017/03/16/spring-vault-1-0-rc1-is-now-available)
-   last week I continued my *Spring Tips* series, this time looking at the reactive HTTP client, the `WebClient`, in preliminary builds of Spring Framework 5 (as used in preliminary builds of Spring Boot 2.0).
-   check out this webinar on domain-driven design by one of [my all-time favorite speakers, InnoQ's Michael Michael Plöd](https://spring.io/blog/2017/03/14/webinar-strategic-domain-driven-design-with-spring-boot)
-   this blog looks at the [`/auditevents` Actuator feature new in Spring Boot 1.5](http://blog.codeleak.pl/2017/03/spring-boot-and-security-events-with-actuator.html)
-   Check out the recording of this webinar by Pivotal's [Daniel Carwin looking at what's happening in the RabbitMQ world](https://www.brighttalk.com/webcast/14883/247741/rabbitmq-what-s-new-changing-after-10-years-of-application-messaging)
-   our friend Christophe F. is back with a continued look [at the new container networking in Pivotal Cloud Foundry and how it performs](https://medium.com/@christophef/container-networking-part2-cloud-foundry-performance-f74f5cc32996)
-   My friend Daniel Bryant has been doing a great job with [his Microservices Weekly news roundup](http://us9.campaign-archive1.com/?id=9a0ad56742&u=3402bc2f02ffba01e245dc556&utm_campaign=buffer&utm_content=bufferf91e7&utm_medium=social&utm_source=twitter.com)
-   Nicolas Fränkel, a friend to the Spring community, is back with an [interesting post in which he tries to use Spring Boot's Actuator in a non-Spring Boot application](https://blog.frankel.ch/fully-configurable-mappings-spring-mvc/#gsc.tab=0). I do *not* recommend this! But, it's a very interesting dissection of what makes the Actuator Framework took. Let Nicolas take you on a journey..
-   I love this blog, [*The Cloud Native Opportunity*](https://content.pivotal.io/blog/the-cloud-native-ops-opportunity)
-   I know this has little to do with the Pivotal ecosystem, but it's a great post from [Adobe that talks about the evolution of the notion of *design*](https://blogs.adobe.com/creativecloud/from-form-to-function-our-thoughts-on-design-are-changing/)
