# Spring Session Hazelcast: Now Led by Hazelcast Team

_Engineering | Rob Winch |  October 14, 2025 | 0 Comments_

It gives me great pleasure to announce that the [Spring Session Hazelcast project will now be led by the Hazelcast Team](https://hazelcast.com/blog/spring-session-hazelcast).

NOTE: This announcement is in alignment with our announcement [Spring Session MongoDB: Now Led by MongoDB Team](https://spring.io/blog/2025/10/14/spring-session-mongodb-new-leadership).

For ten years Spring Session has provided the infrastructure for transparently replacing session stores in a vendor independent manner. With that infrastructure in place, it is just a matter of implementing [SessionRepository](https://docs.spring.io/spring-session/reference/api.html#api-sessionrepository) javadoc in order to support storing session information in a new datastore.

I cannot think of a team who has more Hazelcast experience than the Hazelcast team themselves and so it gives me great pleasure to announce the Hazelcast team will be leading the next generation of Spring Session Hazelcast! To best enable this transition, the next generation of releases will be in a new GitHub repository and have new Maven coordinates. Support for existing generations of Spring Session Hazelcast will continue to be maintained by the Spring Session team as described by the [Spring Support Policy](https://spring.io/support-policy).

For more information, please see the [announcement from the Hazelcast team](https://hazelcast.com/blog/spring-session-hazelcast).
