# This Week in Spring - April 18th, 2017

_Engineering | Josh Long |  April 18, 2017 | 0 Comments_

Hi Spring fans! Welcome to another installment of *This Week in Spring*! This week I'm in Atlanta, GA, at [DevOps Days Atlanta](https://www.devopsdays.org/events/2017-atlanta/speakers/josh-long/), and Boston for the [Agile Tech Alliance conference](https://www.agilealliance.org/agile-alliance-technical-conference-2017/). If you're around for either I'd love to see you - [say hi](http://twitter.com/starbuxman)!

Don't miss me, Spring Team members, and community speakers at the upcoming Spring Days [Chicago](https://www.springdays.io/ehome/spring-days/chicago), [New York](https://www.springdays.io/ehome/spring-days/new-york) and [Atlanta](https://www.springdays.io/ehome/spring-days/atlanta) events!

As usual, we've got a lot to cover so let's get to it!

-   The WebFlux functional reactive support in Spring Framework 5.0 is really catching on! I particularly enjoyed this post called [*Spring: Your Next Microframework*](http://blog.alexnesterov.com/post/spring-your-next-microframework/)
-   Baeldung has a great post introducing [WebFlux Functional Reactive Endpoints](http://www.baeldung.com/spring-5-functional-web)
-   Jonathan Borenstein put together a simple demonstration [looking at building functional REST services with Spring Framework 5.0](https://github.com/jonathanborenstein/reactive-demo)
-   Hurray! [The Spring Cloud Dalston release is now GA!](https://spring.io/blog/2017/04/12/spring-cloud-dalston-released) Get the bits and try it out! This release is *packed* with great new stuff!
-   Spring Cloud Task ninja Glenn Renfro [just announced Spring Cloud Task 1.2.0.RC1](https://spring.io/blog/2017/04/14/spring-cloud-task-1-2-0-rc1-is-now-available)
-   Spring Cloud Stream lead Marius Bogoevici [just announced Spring Cloud Stream Chelsea GA](https://spring.io/blog/2017/04/13/spring-cloud-stream-chelsea-goes-ga)
-   Spring Data lead [Oliver Gierke just announced Spring Data Kay M2](https://spring.io/blog/2017/04/13/spring-data-kay-m2-released) with *lots* of improvements and bugfixes! Get it while it's hot!
-   last week I concluded [season 2 of the Spring Tips series](https://spring.io/blog/2017/04/12/spring-tips-season-2-recap). Read on for a review of the installments from the last 11 episodes and more!
-   Spring Cloud Vault lead [Mark Paluch just announced Spring Cloud Vault 1.0](https://spring.io/blog/2017/04/12/spring-vault-1-0-goes-ga)
-   I did a podcast with my buddy and colleague [Michael Coté](http://www.cote.show/23) on [enabling microservices with devops](http://www.cote.show/23). Or something.
-   periodic reminder that [SDKMAN!](http://sdkman.io/sdks.html) is awesome and you can use it to install Spring Boot: `sdk install springboot`
-   Using the JVM? (I sure hope so!) You might care about [the discussion of the deprecation of `java.lang.Object#finalize`](https://stuartmarks.wordpress.com/2017/04/17/deprecation-of-object-finalize/)
-   Tangent: ZeroTurnaround put out a great [*cheat sheet* covering some tips-and-tricks with `git` last year](https://zeroturnaround.com/rebellabs/git-commands-and-best-practices-cheat-sheet/)
-   the Java Pub House podcast has a nice interview [with Spring Batch and Spring Cloud Task lead Michael Minella](http://www.javapubhouse.com/2017/04/episode-64-oh-its-time-to-re-boot.html)
-   Łukasz Monkiewicz put together a nice post [on request validation with Spring Boot](https://lmonkiewicz.com/programming/get-noticed-2017/spring-boot-rest-request-validation/)
-   Friendly remminder: the goal behind microservices, cloud-computing, agile and everything else you'll hear Pivotal talking about is, ultimately, to get [to continuous delivery, some of the tenants of which are nicely explained here](https://technologyconversations.com/2017/03/06/the-ten-commandments-of-continuous-delivery/)
-   Lars Trieloff, from Adobe, put together a very cool post discussing the paradigm shift in REST vs technologies like webhooks, where you're still doing REST but they become more [like topics in a messaging architecture](https://medium.com/adobe-io/what-event-based-architectures-mean-for-application-integration-662932169d28)
-   I love this post, [*RabbitMQ: My Alternative ServiceBus for Windows Server*](https://www.robfox.io/2017/04/17/rabbitmq-alternative-service-bus-windows-server/)
-   Speaking of RabbitMQ: have you seen [the new RabbitMQ website? It's dynamite!](http://www.rabbitmq.com/)
-   Pivotal Cloud Foundry now [has built-in support for durable persistence](https://content.pivotal.io/blog/apps-that-depend-on-file-storage-bring-em-over-to-pivotal-cloud-foundry)
-   I liked this writeup of Neoj chief scientist Jim Webber's introduction [to how Neo4j achieves cluster consensus](https://www.infoq.com/news/2017/04/causal-consistency-neo4j)
-   The Cloud Technology Partners podcast had a great interview with Pivotal's [Richard Seroter looking at the future of Cloud Foundry](https://www.cloudtp.com/doppler/best-practices/whats-new-cloud-foundry-pivotal/)
-   JaxEnter has a great German-language survey on microservices trend\]([https://jaxenter.de/microservices-trends-55937](https://jaxenter.de/microservices-trends-55937))
-   Baeldung also has a great post on using [the `@JsonComponent` annotation with Spring Boot](http://www.baeldung.com/spring-boot-jsoncomponent)
