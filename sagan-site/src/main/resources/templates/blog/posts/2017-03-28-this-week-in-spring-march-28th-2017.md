# This Week in Spring - March 28th, 2017

_Engineering | Josh Long |  March 28, 2017 | 0 Comments_

Welcome to another installment of *This Week in Spring*! This week I'm in Omaha, Nebraska, speaking to customers and tonight I'll be presenting at an event co-sponsored by the Omaha, JUG *and* the local Omaha, Spring Users Group, [talking about reactive Spring](https://www.meetup.com/omahajava/events/237784204/). Don't be shy, come out [and say hi](http://twitter.com/starbuxman)!

-   Sébastien Deleuze just introduced the 1.0.0.M1 relese of extensions [supporting using Project Reactor in the Kotlin language](https://spring.io/blog/2017/03/28/reactor-kotlin-extensions-1-0-0-m1-released)
-   Spring REST Docs lead [Andy Wilkinson just announced](https://spring.io/blog/2017/03/24/spring-rest-docs-1-2-0-rc1) Spring REST Docs 1.2.0.RC1. Try it out now! For a good example of Spring REST Docs, scroll down and you'll see a very nice one from a community member.
-   Spring Cloud Data Flow contributor and all-around great human Thomas Risberg

[just announced Spring Cloud Data Flow 1.2.M3](https://spring.io/blog/2017/03/22/spring-cloud-data-flow-1-2-m3-released)

-   last week, continuing the Spring Tips series, I looked at the various [options for deploying Spring Boot applications to Servlet containers](https://spring.io/blog/2017/03/22/spring-tips-servlet-container-deployment-options)
-   Going beyond the Spring Boot intelligent defaults? Need to debug to that next level? Join Nicolas Fränkel, Hybris as he takes us [under the hood of Spring Boot](https://spring.io/blog/2017/03/28/webinar-spring-boot-under-the-hood) on Thursday April 13th.
-   it's still not too late to register for wednesday's webinar with Michael Plod (InnoQ) on [Strategic (Domain Driven) Design with Spring Boot](https://spring.io/blog/2017/03/14/webinar-strategic-domain-driven-design-with-spring-boot)
-   Spring Cloud co-founder Spencer Gibb is at it again, this time working [on a new Gateway project called Spring Cloud Gateway](https://github.com/spring-cloud-incubator/spring-cloud-gateway). This project is nowhere near finished or final, but now's a great time to get the bits, kick the tires and feedback. Spencer's been working hard adding all sorts of cool things, [like request throttling](https://github.com/spring-cloud-incubator/spring-cloud-gateway/commit/0105e57ac8bf198795689ef4d6bc592794ddb6e9).
-   there are still some very interesting things that you can do with Netflix's Zuul, too! If you want to learn more, [check out the ever-expanding repository full of samples](https://github.com/spring-cloud-samples/sample-zuul-filters)
-   [Jenn Strater](http://twitter.com/codeJENNerator) did a really nice job [demonstrating Spring RESTDocs](https://github.com/jlstrater/groovy-spring-boot-restdocs-example) in this example.
-   Finally! a proposal to [deprecate (but of course never absolutely remove) Java's finalization behavior](https://www.infoq.com/news/2017/03/Java-Finalize-Deprecated)
-   My training course, [Cloud Native Java, is now available on Safari](https://www.safaribooksonline.com/library/view/cloud-native-java/9780134690377/) and for purchase.
-   I liked the most recent installment of [*Pivotal Conversations* which discusses the CRE program](https://soundcloud.com/pivotalconversations/running-like-google-the-cre-program-pivotal-with-andrew-shafer) speared by Pivotal and Google to ensure that Pivotal Cloud Foundry on Google Cloud Platform delivers a Google-level of reliability
-   Venkat Rangasamy has an interesting post [on Production-Ready Microservices](https://dzone.com/articles/production-ready-microservices-in-action) that talks about, but doesn't focus on, Spring Boot
-   I liked this Italian-language [post by Federico Paparoni on using the Spring Boot Actuator for metrics](https://www.javastaff.com/2017/03/27/metriche-con-spring-boot-actuator/)
-   Pivotal's own Richard Seroter looks at using a single service registry to talk [to coordinate both .NET and Spring Cloud-based Microservices](https://seroter.wordpress.com/2017/03/27/yes-you-can-use-a-single-service-registry-for-net-and-java-microservices/)
-   I *also* liked this Federico's Italian-language post [on supporting CRUD with Spring Data REST and Spring Data JPA](http://www.javastaff.com/2017/03/24/crud-con-spring-boot-rest-jpa/)
