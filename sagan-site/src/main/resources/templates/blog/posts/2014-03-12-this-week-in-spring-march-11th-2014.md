# This Week in Spring - March 11th, 2014

_Engineering | Josh Long |  March 12, 2014 | 0 Comments_

Welcome to another installment of *This Week in Spring*! As usual, we've got a lot to cover, so let's get to it!

1.  Spring AMQP lead Gary Russell has just announced [that Spring AMQP 1.3.0 has just been released](http://spring.io/blog/2014/03/07/spring-amqp-1-3-0-release-candidate-available)!
2.  Spring Social lead Craig Walls just announced the [first, long-awaited, RC of Spring Social 1.1](http://spring.io/blog/2014/03/11/spring-social-1-1-0-rc1-released). Hurray! I'm excited to see this new release.
3.  Spring Security lead Rob Winch has just announced [a security fix for Spring Security 3.2.2 and 3.1.6](http://spring.io/blog/2014/03/11/cve-2014-009-fixed-in-spring-security-3-2-2-and-3-1-6).
4.  Dr. Dave Syer, co-lead on Spring Boot, among many, many other amazing things, has thrown together a sexy looking [Spring Boot wrapper for Ratpack, a toolkit for JVM web applications](https://github.com/dsyer/spring-boot-ratpack). That's *so* awesome.
5.  Spring Data lead Oliver Gierke has just announced that the first service release [for Spring Data Babbage is now available](http://spring.io/blog/2014/03/10/final-service-release-for-spring-data-release-train-babbage)
6.  I put together a post on [the various deployment strategies for Spring Boot applications](http://spring.io/blog/2014/03/07/deploying-spring-boot-applications)
7.  Don't miss Mattias Arthursson on a webinar March 18th, presenting [Spring LDAP 2.0.0](http://spring.io/blog/2014/02/11/webinar-spring-ldap-2-0-0-on-march-18-2014).
8.  Join Juergen Hoeller and the Spring Team for a webinar on [Java 8 and Spring Framework 4.0](https://spring.io/blog/2014/02/11/webinar-spring-framework-4-0-on-java-8-march-25) on March 25!
9.  Speaking of Spring Data, check out the [replay of this talk introducing Spring Data repository best practices from SpringOne2GX 2013](http://spring.io/blog/2014/03/05/webinar-replay-spring-data-repositories-best-practices)
10.  Don't miss John Hann's introduction to [pragmatic JavaScript from SpringOne2GX 2013](http://spring.io/blog/2014/03/11/springone2gx-2013-replay-pragmatic-javascript)
11.  Emanuel Rabina gave a [talk at SpringOne2GX 2013 on improving your Spring view layer with natural templates and Thymeleaf](http://spring.io/blog/2014/03/11/springone2gx-2013-replay-thymeleaf-improving-your-spring-view-layer-with-natural-templates): check it out!
12. 13.  In related news, snapshots of the Vaadin4Spring [integration that Petter and I are working on are now published on Sonatype's repository](https://oss.sonatype.org/content/repositories/snapshots/org/vaadin/spring/) - enjoy!
14.  Our pal Eberhard Wolff is back, and this time he's written up a nice (German-language) post on [Spring 4.0 and Spring Boot, among (many) other things](http://www.heise.de/developer/artikel/Der-Stand-der-Dinge-bei-Spring-Spring-4-0-Spring-Boot-und-Spring-XD-2139704.html)
15.  David Williams blog, [*A Time-to-Value Story with Cloud Foundry*, is thought-provoking](http://www.williamsgarcia.com/2014/03/07/a-time-to-value-story-with-cloud-foundry/): how quickly can a technology deliver *value* - measurable value?
16.  Martin Fowler has started [watching the micro services space](http://martinfowler.com/articles/microservices.html). Nothing to report, specifically. This is just a page I'd recommend watching.
17.  Layer 7, a CA technologies company, has just released results of a survey focused on API design and deployment. There are many takeaways from the survey, so be sure to read it, but one thing I thought particularly promising: [a predicted growth in Hypermedia-aware APIs](http://www.infoq.com/news/2014/03/ca-api-survey) among API developers. If you're interested in embracing hypermedia, and the design pattern HATEOAS, I would encourage [you to check out Spring HATEOAS](http://spring.io/projects/spring-hateoas) and - if you'd like to learn more about using Spring HATEOAS and Spring Boot, [check out this tutorial and adjoining code](https://github.com/joshlong/the-spring-rest-stack).
