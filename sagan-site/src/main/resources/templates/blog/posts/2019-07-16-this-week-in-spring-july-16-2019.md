# This Week in Spring - July 16, 2019

_Engineering | Josh Long |  July 16, 2019 | 1 Comment_

HI Spring fans! Welcome to another installment of *This Week in Spring*! I've just returned from Lima, Peru, for JConf Peru, and then today finished speaking at the SpringOne Tour Philadelphia event. Now, I'm off to Portland, OR, for the epic OSCON show. As usual, we've got tons to cover so let's get to it!

-   [Spring Tools 4.3.1 released](https://spring.io/blog/2019/07/12/spring-tools-4-3-1-released)
-   In last week's installment of [*A Bootiful Podcast*, I interview Mr. REST / HATEOAS, Mike Amundsen, on APIs, HTTP, REST and more](https://spring.io/blog/2019/07/12/a-bootiful-podcast-mike-amundsen-on-apis-http-rest-and-more)
-   This is an interesting post [on how Orange makes use of Cloud Foundry](https://www.cloudfoundry.org/blog/orange-makes-novel-use-of-cloud-foundry-for-best-developer-experience/)
-   [Check out this resolution for those using Spring Cloud Gateway and experiencing issues doing a retry `POST`](https://github.com/spring-cloud/spring-cloud-gateway/issues/1160)
-   Check out [the RSocket YouTube channel](https://medium.com/netifi/rsocket-video-channel-on-youtube-d33f1562a493)
-   Have you registered for SpringOne Platform 2019? [Do it](https://springoneplatform.io/register?utm_campaign=s1p-website-social&utm_content=1561151675&utm_medium=social-sprout&utm_source=twitter)
-   This is amazing - [a new reactive Spring Data Neo4J implementation](https://github.com/neo4j/sdn-rx) is coming!
-   Check this out - [there's now a Kubernetes-ready distribution of RabbitMQ](https://content.pivotal.io/blog/introducing-rabbitmq-for-kubernetes). *Now* it's cloud-native!
-   I love the new Microsoft Azure API development guidelines which encourage the use of Project Reactor [to build elegant, asynchronous, reactive APIs](https://azuresdkspecs.z5.web.core.windows.net/JavaSpec.html)
-   Here's a nice post by JHipster lead and Azure Developer Advocate Julien Dubois on going [fully reactive with Spring Webflux and the new CosmosDB API v3](https://dev.to/azure/going-full-reactive-with-spring-webflux-and-the-new-cosmosdb-api-v3-1n2a)
