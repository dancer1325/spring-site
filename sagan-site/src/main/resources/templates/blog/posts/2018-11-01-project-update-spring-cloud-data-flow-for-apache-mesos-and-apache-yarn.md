# Project Update: Spring Cloud Data Flow for Apache Mesos and Apache YARN

_News | Mark Pollack |  November 01, 2018 | 0 Comments_

Dear Spring community,

The Spring Cloud Data Flow team have been happy stewards of the Spring Cloud Deployer and Spring Cloud Data Flow implementations of [Apache Mesos](https://github.com/spring-cloud?utf8=%E2%9C%93&q=mesos&type=&language=) and [Apache YARN](https://github.com/spring-cloud?utf8=%E2%9C%93&q=yarn&type=&language=).

We now feel that [TrustedChoice.com](https://trustedchoice.com) will be a better home for Apache Mesos implementation of Spring Cloud Deployer and Spring Cloud Data Flow, and we are donating the projects to them to carry it forward.

The development will now be managed directly by the team ([Adam J. Weigold](https://github.com/aweigold), [Phil Egelston](https://github.com/pegelston), [Justin Mathieu](https://github.com/Jmathieu3289), and [Cole Anderson](https://github.com/bitskeptic-tc)) at TrustedChoice.com, as the Spring Cloud Data Flow team will no longer maintain it.

For more details, please refer to the projects in TrustedChoice.com’s GitHub repositories:

-   [Spring Cloud Deployer for Mesos](https://github.com/trustedchoice/spring-cloud-deployer-mesos)
    
-   [Spring Cloud Data Flow for Mesos](https://github.com/trustedchoice/spring-cloud-dataflow-server-mesos)
    

For Apache YARN, however, since we are focusing our efforts towards Local, Kubernetes and Cloud Foundry implementations, the Spring Cloud Data Flow team has stopped maintaining it. The Apache Yarn implementation of Spring Cloud Deployer and Spring Cloud Data Flow projects have officially reached the end-of-life (EOL) status today (November 1st, 2018). One year from the announced date (i.e., November 2nd, 2019), the projects will officially reach the end-of-general support (EOGS).

If there’s interest in the community, we would like to open it up and donate the related projects similar to Apache Mesos implementation. If you are interested in forking the project to continue to develop and maintaining it, we would be happy to cross-link and promote your forked project. Please ping us at [Gitter](https://gitter.im/spring-cloud/spring-cloud-dataflow) to discuss it.

All the best,

Spring Cloud Data Flow Team
