# Spring Cloud Brixton.SR5 is available

_Releases | Ryan Baxter |  August 23, 2016 | 0 Comments_

On behalf of the team, I am pleased to announce that Service Release 5 of the [Spring Cloud Brixton](http://cloud.spring.io) Release Train is available today. The release can be found in our [Spring Release repository](http://repo.spring.io/release) and [Maven Central](http://search.maven.org/#search%7Cga%7C1%7Ca%3A%22spring-cloud-dependencies%22).

This release includes primarily bug fixes.

This release also deprecates [Spring Cloud Cluster](https://github.com/spring-cloud/spring-cloud-cluster) in favor of [Spring Integration](https://projects.spring.io/spring-integration/).

#### [](#the-following-modules-were-updated-as-part-of-brixtonsr5)The following modules were updated as part of Brixton.SR5:

-   Spring Cloud Bus [1.1.1.RELEASE](https://github.com/spring-cloud/spring-cloud-bus/milestone/13?closed=1)
-   Spring Cloud Config [1.1.3.RELEASE](https://github.com/spring-cloud/spring-cloud-config/milestone/21?closed=1)
-   Spring Cloud AWS [1.1.1.RELEASE](https://github.com/spring-cloud/spring-cloud-aws/milestone/12?closed=1)
-   Spring Cloud Sleuth [1.0.6.RELEASE](https://github.com/spring-cloud/spring-cloud-sleuth/milestone/12?closed=1)
-   Spring Cloud Task [1.0.2.RELEASE](https://github.com/spring-cloud/spring-cloud-task/milestone/7?closed=1)

And, as always, we welcome feedback: either on [GitHub](https://github.com/spring-cloud), on [gitter](https://gitter.im/spring-cloud/spring-cloud), on [Stack Overflow](http://stackoverflow.com/questions/tagged/spring-cloud), or on [Twitter](https://twitter.com/SpringCloudOSS).
