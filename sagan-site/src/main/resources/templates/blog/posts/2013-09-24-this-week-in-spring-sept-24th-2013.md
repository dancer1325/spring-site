# This Week in Spring - Sept 24th 2013

_News | Pieter Humphrey |  September 24, 2013 | 0 Comments_

Hello everyone! Pieter posting this on Josh Long's behalf here... this week the Spring and Cloud Foundry teams are at JavaOne talking to developers about Spring, Cloud Foundry, Hadoop and big data! Happily, our pretty new [@GoPivotal](http://pivotal.com) booth has garnered a lot of attention!

![](https://pbs.twimg.com/media/BU4S8CvCcAAVchk.jpg)

I met Duke, too! I think the big guy was as happy to see me as I was him!

![](https://pbs.twimg.com/media/BU-9EONCYAAKuFt.jpg)

1.  Spring ninja Greg Turnquist put together a nice post on [how to contribute to Spring Boot with a pull request](https://spring.io/blog/2013/09/20/contributing-to-spring-boot-with-a-pull-request). The article *does* ultimately talk about the processing sending a pull request, but most of it's a really great introduction to extending Spring Boot to suit a new use case (like JMS, which simply hadn't been added yet). This post also introduces some cool features in Spring Boot, if you haven't learned about it yet.
2.  Speaking of Spring Boot, check out [this upcoming Spring Boot webinar featuring project co-lead Phil Webb](https://spring.io/blog/2013/09/19/webinar-simplifying-spring-with-spring-boot)!
3.  Rossen Stoyanchev put together a *knock-out* piece how to build [web applications that use websockets, or Sock.js, and how to then layer STOMP](https://spring.io/blog/2013/09/24/webinar-building-websocket-browser-applications-with-spring) on top of that for an easier messaging-style interaction between the browser and the service.
4.  Rossen also has an upcoming webinar on Oct 8th, [Building WebSocket Browser Applications with Spring](https://spring.io/blog/2013/09/24/webinar-building-websocket-browser-applications-with-spring), showing off the best of his SpringOne2GX 2013 session.
5.  I encourage you to check out [the upcoming webinar featuring James Williams on building big data applications with Spring, Spring XD and Hadoop](https://spring.io/blog/2013/09/24/webinar-building-big-data-applications-with-hadoop-spring)! This is going to be an awesome talk. James has some killer examples and knows his stuff.
6.  In related news, I put together a quick post on my thoughts [on Data Integration, Stream Processing and Spring XD](http://www.joshlong.com/jl/blogPost/data_integration_stream_processing_and_spring_xd.html).
7.  Jakub Kubrynski has put together a nice post on how to use load-time weaving to [inject dependencies into objects *not* managed by the Spring container using a couple of different approaches](http://www.javacodegeeks.com/2013/09/injecting-spring-beans-into-non-managed-objects.html).
8.  Nitin Kumar put together [a nice look at Spring Integration, a lightweight approach](http://java.dzone.com/articles/spring-integration-lightweight).
9.  Tomás Lin, who says that he loves testing Grails applications with Spock, put together a [great post on how to test with Spring Boot with Spock](http://fbflex.wordpress.com/2013/09/18/testing-spring-boot-applications-with-spock/).
10.  Sometimes, you need to do RMI, and when you do, [Spring has you covered](http://sethuramanmurali.wordpress.com/2013/09/20/rmi-calls-using-spring/)!
11.  The *BitwiseOR* blog has a *great* post on [how to use Spring Boot to build REST services, using Groovy no less](http://bitwiseor.com/2013/09/20/creating-test-services-with-spring-boot/)! Win! I'm not sure if I agree with the conclusion, but *whatever* it takes for people to be able to use this to simplify development, it's a win! And, with really great posts like this, hopefully it won't be long at all before everyone's *boot*strapping their development, too!
