# Spring Session for Apache Geode & Pivotal GemFire 2.1.6.RELEASE and 2.2.0.RC1 Available

_Releases | John Blum |  October 02, 2019 | 0 Comments_

On behalf of the Spring and Apache Geode & Pivotal GemFire communities, it is my pleasure to announce the release of *Spring Session for Apache Geode & Pivotal GemFire* (SSDG) `2.1.6.RELEASE` and `2.2.0.RC1`.

SSDG `2.1.6.RELEASE` is available in [Maven Central](https://search.maven.org/search?q=a:spring-session-data-geode) and SSDG `2.2.0.RC1` can be acquired from the Spring [Milestone](https://repo.spring.io/milestone/org/springframework/session/spring-session-data-geode/2.2.0.RC1/) Repository.

Both SSDG `2.1.6.RELEASE` and `2.2.0.RC1` are included in the Spring Session `Bean-SR9` and `Corn-RC1` BOM files, which will be picked up by Spring Boot `2.1.9.RELEASE` and the Spring Boot `2.2.0.RC1` releases, respectively.

Additionally, SSDG `2.1.6.RELEASE` will be picked up by the *Spring Boot for Apache Geode* (SBDG) `1.1.2.RELEASE` and SSDG `2.2.0.RC1` will be picked up by Spring Boot for Apache Geode `1.2.0.RC1`.

## [](#whats-new)[](#whats-new)What’s New

SSDG `2.1.6.RELEASE` primarily includes dependency updates:

-   Spring Framework `5.1.10.RELEASE`
    
-   Spring Data `Lovelace-SR11`
    
-   Spring Session `2.1.9.RELEASE`
    

SSDG `2.2.0.RC1` primarily includes dependency updates:

-   Spring Framework `5.2.0.RELEASE`
    
-   Spring Data `Moore-RELEASE`
    
-   Spring Session `2.2.0.RC1`
    

## [](#whats-next)[](#whats-next)What’s Next

In the coming months SSDG will align with some new improvements in Spring Session, such as, but not limited to:

-   Support for the `SessionRepositoryCustomizer` ([Issue #42](https://github.com/spring-projects/spring-session-data-geode/issues/42))
    
-   Support for a "*flush*" mode ([Issue #44](https://github.com/spring-projects/spring-session-data-geode/issues/44) / ([Issue #13](https://github.com/spring-projects/spring-session-data-geode/issues/13))
    
-   Support for the new `IndexResolver` interface ([Issue #41](https://github.com/spring-projects/spring-session-data-geode/issues/41))
    

In addition, SSDG 2.3 will see some harmonizing around naming in the API ([Issue #40](https://github.com/spring-projects/spring-session-data-geode/issues/40)).
