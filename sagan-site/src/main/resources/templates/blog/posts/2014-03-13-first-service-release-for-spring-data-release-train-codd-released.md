# First service release for Spring Data release train Codd released

_Releases | Oliver Drotbohm |  March 13, 2014 | 0 Comments_

I am happy to announce the first service release of the Spring Data release train named [Codd](http://en.wikipedia.org/wiki/Edgar_F._Codd). The full list of participating modules looks as follows:

-   Spring Data Build 1.3.1 - [Changelog](https://github.com/spring-projects/spring-data-build/issues?milestone=14&state=closed)
-   Spring Data Commons 1.7.1 - [Artifacts](http://repo.spring.io/libs-milestone/org/springframework/data/spring-data-commons/1.7.1.RELEASE/) - [JavaDoc](http://docs.spring.io/spring-data/commons/docs/1.7.1.RELEASE/api/) - [Documentation](http://docs.spring.io/spring-data/commons/docs/1.7.1.RELEASE/reference/htmlsingle/) - [Changelog](http://docs.spring.io/spring-data/commons/docs/1.7.1.RELEASE/changelog.txt)
-   Spring Data JPA 1.5.1 - [Artifacts](http://repo.spring.io/libs-milestone/org/springframework/data/spring-data-jpa/1.5.1.RELEASE/) - [JavaDoc](http://docs.spring.io/spring-data/jpa/docs/1.5.1.RELEASE/api/) - [Documentation](http://docs.spring.io/spring-data/jpa/docs/1.5.1.RELEASE/reference/htmlsingle/) - [Changelog](http://docs.spring.io/spring-data/jpa/docs/1.5.1.RELEASE/changelog.txt)
-   Spring Data MongoDB 1.4.1 - [Artifacts](http://repo.spring.io/libs-milestone/org/springframework/data/spring-data-mongodb/1.4.1.RELEASE/) - [JavaDoc](http://docs.spring.io/spring-data/mongodb/docs/1.4.1.RELEASE/api/) - [Documentation](http://docs.spring.io/spring-data/mongodb/docs/1.4.1.RELEASE/reference/htmlsingle/) - [Changelog](http://docs.spring.io/spring-data/mongodb/docs/1.4.1.RELEASE/changelog.txt)
-   Spring Data Neo4j 3.0.1 - [Artifacts](http://repo.spring.io/libs-milestone/org/springframework/data/spring-data-neo4j/3.0.1.RELEASE/) - [JavaDoc](http://docs.spring.io/spring-data/neo4j/docs/3.0.1.RELEASE/api/) - [Documentation](http://docs.spring.io/spring-data/neo4j/docs/3.0.1.RELEASE/reference/htmlsingle/) - [Changelog](http://docs.spring.io/spring-data/neo4j/docs/3.0.1.RELEASE/changelog.txt)
-   Spring Data Solr 1.1.1 - [Artifacts](http://repo.spring.io/libs-milestone/org/springframework/data/spring-data-solr/1.1.1.RELEASE/) - [JavaDoc](http://docs.spring.io/spring-data/solr/docs/1.1.1.RELEASE/api/) - [Documentation](http://docs.spring.io/spring-data/solr/docs/1.1.1.RELEASE/reference/htmlsingle/) - [Changelog](http://docs.spring.io/spring-data/solr/docs/1.1.1.RELEASE/changelog.txt)
-   Spring Data REST 2.0.1 - [Artifacts](http://repo.spring.io/libs-milestone/org/springframework/data/spring-data-rest-webmvc/2.0.1.RELEASE/) - [JavaDoc](http://docs.spring.io/spring-data/rest/docs/2.0.1.RELEASE/api/) - [Documentation](http://docs.spring.io/spring-data/rest/docs/2.0.1.RELEASE/reference/htmlsingle/) - [Changelog](http://docs.spring.io/spring-data/rest/docs/2.0.1.RELEASE/changelog.txt)

For this release we've squashed [42 tickets](https://jira.spring.io/issues/?filter=14543). As the release contains a few important bug fixes it is a recommended upgrade for all Codd users. Also, users of the previous release train Babbage are encouraged to upgrade.

The next release following in two week will be the first milestone of our nexte release train called [Dijkstra](http://en.wikipedia.org/wiki/Edsger_W._Dijkstra). We'll have quite a few new modules joining the release train. A preliminary schedule is available at the [wiki page](https://github.com/spring-projects/spring-data-commons/wiki/Release-Train-Dijkstra) for the release. We're currently in the planning phase and will happily consider the input you cast in JIRA tickets.
