# Video: SpringOne 2GX - Tuning Java for Virtual Environments with EM4J

_News | Adam Fitzgerald |  February 16, 2012 | 0 Comments_

[![SpringOne 2GX Video: Tuning Java for Virtual Environments with EM4J](http://www.springsource.org/files/SpringOne2GX2012TuningJavaForVirtualEnvironment.png)](http://www.infoq.com/presentations/Tuning-Java-for-Virtual-with-EM4J)

As datacenters virtualize more and more workloads, it becomes important to understand the potential impact on the Java servers that may be running your applications. At SpringOne 2GX 2011, [Ben Corrie](http://blog.springsource.org/author/bcorrie/) provided a thorough [overview of Java memory management](http://www.infoq.com/presentations/Tuning-Java-for-Virtual-with-EM4J), how things work differently in virtual environments and what you can do about it to avoid problems. This includes a practical introduction to Elastic Memory for Java (EM4J) that is a feature of the [VMware vFabric platform](http://www.vmware.com/products/application-platform/vfabric/overview.html).

Many thanks to [InfoQ](http://www.infoq.com) for coming to Chicago to record so many of the fantastic [SpringOne 2GX presentations](http://www.infoq.com/springone_2gx_2011/).
