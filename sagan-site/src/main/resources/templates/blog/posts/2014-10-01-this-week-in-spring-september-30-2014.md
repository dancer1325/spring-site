# This Week in Spring - September 30, 2014

_Engineering | Josh Long |  October 01, 2014 | 3 Comments_

Welcome to another installment of *This Week in Spring!* This week [the Spring team is at JavaOne](http://spring.io/blog/2014/09/26/pivotal-at-javaone-2014)! There's been, and will be, *lots* to see! We're also hanging out at the booth, talking to as many users as possible. Check out the link I just give listing all the cool Pivotal (and non-Pivotal) talks at JavaOne. (Ahem. Naturally, I personally would *love* it if you made [my talk - *Spring4TW!* - tomorrow morning at 8:30am at Parc 55, Cyril Magnin II/III](http://oracleus.activeevents.com/2014/connect/sessionDetail.ww?SESSION_ID=3693&tclass=popup)!).

![](https://raw.githubusercontent.com/joshlong/meh/master/javaone2014-boot-twis.jpg)

In addition to IoT, Big Data, and Microservice demos using [NetFlixOSS on Spring](http://projects.spring.io/spring-cloud/), students were doing the coding challenge to build a [REST service with Spring Boot](https://spring.io/guides/gs/rest-service/) and pushed to [Pivotal Web Services](http://run.pivotal.io) in < 15 minutes to get the [preshavedyak hoodie](http://preshavedyak.com/) from American Apparel, and learning about the best PaaS for Java available on the market today; Java 8 / Tomcat 8 support was released just days ago, and there is strong support for Spring, Groovy and Grails.

-   I really liked Michael Plod's [*Why I Recommend Spring* webinar](http://spring.io/blog/2014/09/26/webinar-replay-why-i-recommend-spring), now available online as a replay
-   Last week, Activiti BPMN2 engine co-founder Joram Barrez and I gave a talk on [how to build process (read: workflow) centric Spring applications](http://spring.io/blog/2014/09/26/webinar-replay-process-driven-spring-applications-with-activiti), also now available online.
-   Tuesday, Oct 21st, 2014 join [Matt Stine](https://twitter.com/mstine) for microservice architecture series #2: [Architecting for Continuous Delivery: Microservices with Pivotal CF and Spring Cloud](http://spring.io/blog/2014/10/01/webinar-architecting-for-continuous-delivery-microservices-with-pivotal-cf-and-spring-cloud)
-   Thursday, October 23, 2014, 2014 [Building “Bootful” UIs with Spring Boot and Vaadin](http://spring.io/blog/2014/10/01/webinar-building-bootful-uis-with-spring-boot-and-vaadin)
-   Tuesday, November 4, 2014 - microservice architecture series #3 - Join [Dr. Dave Syer](https://twitter.com/david_syer) on [Security for Microservices with Spring and OAuth2](http://spring.io/blog/2014/10/01/webinar-security-for-microservices-with-spring-and-oauth2)
-   Missed microservice architecture #1? Catch the replay of [Building "Bootiful" Microservices with Spring Boot](http://spring.io/blog/2014/09/17/webinar-replay-building-bootiful-microservices-with-spring-boot) on the Spring Developer Channel
-   [Spring Boot 1.1.7 is now available](http://spring.io/blog/2014/09/26/spring-boot-1-1-7-released)!
-   [Setup Spring Security with Active Directory LDAP in Spring Boot Web Application](http://raymondhlee.wordpress.com/2014/09/20/setup-spring-security-with-active-directory-ldap-in-spring-boot-web-application/) (I *just* got asked about this at the Spring BOF yesterday!)
-   There's been an interesting debate on microservices recently. ["Uncle" Bob Martin has written up a post claiming that `.jar`s and Ruby `gem`s give you the same benefit of microservices](http://blog.cleancoder.com/uncle-bob/2014/09/19/MicroServicesAndJars.html). There's a [great rebuttal (which I happen to agree with..) by Giorgio Sironi on his blog](http://www.giorgiosironi.com/2014/09/microservices-are-not-jars.html). There's even a nice wrapup of [the whole discussion on InfoQ](http://www.infoq.com/news/2014/09/microservices-shared-libraries?utm_source=infoq&utm_medium=popular_links_homepage).
-   I really dig [Matt Stine's *wind tunnel* comparison of PaaSes](http://blog.pivotal.io/cloud-foundry-pivotal/case-studies-2/platform-wind-tunnel-part-i-heroku). Up first: Heroku. I can't wait to see subsequent posts in the series!
-   This post on using Spring Data Redis was published (and included in *TWiS*..) last year, [but it came up again and I still thnk it's a worthy read](http://www.ibm.com/developerworks/library/os-springredis/index.html)
-   The eHarmony blog has a nice post on [how to setup Spring Batch Admin (the tricky parts..)](http://www.eharmony.com/engineering/spring-batch-admin-the-tricky-parts/#.VCtPFildUpg)
-   In the *you have to see it to beleive it* department, did you know there's a [Spring for Delphi (Spring4D) project](http://bitbucket.org/sglienke/spring4d)? There are Spring-inspired C++, ColdFusion, JavaScript, .NET, Python, and numerous others. Cool to add this to the list.
-   The Intertech blog has [another installment on their *epic* Spring Integration tutorial](http://www.intertech.com/Blog/spring-integration-tutorial-part-7-service-activators/). This one's on *service activators* - basically the Spring Integration escape-hatch so you can call regular Java objects.
-   Benjamin Ball has a great [writeup on using RabbitMQ and Spring](http://java.dzone.com/articles/getting-started-rabbitmq-and)
-   Yashwant Chavan has a nice post on building a simple [Spring Boot (with Spring MVC) example and editing it in Eclipse](http://www.technicalkeeda.com/spring/spring-boot-mvc-example).
-   Our pals at Codecentric are at it again! They've just released [their first cut of their *Spring Boot Admin*](http://blog.codecentric.de/en/2014/09/spring-boot-admin-first-official-release/). This looks like something that would fit nicely in the [Spring Cloud](http://github.com/spring-cloud/) project...
-   [This has to be one of the coolest Spring Boot presentations, ever](http://prezi.com/ob4vxjmj_h1f/spring-boot/)! Careful, I don't recommend watching it full-screen if you get motion sickness! :)
-   Are you stuck on Apache CXF but still want first-class security? This post [looks at how to plugin Spring Security on top of CXF in minutes](http://java.dzone.com/articles/embedded-jetty-and-apache-cxf)
-   Biju Kunjummen did a nice job [clarifying the differences between Spring's `WebApplicationInitializer` and `ApplicationContextInitializer`](http://java.dzone.com/articles/spring-1)
