# Spring Data 2025.0.0-M1 released

_Releases | Mark Paluch |  February 14, 2025 | 0 Comments_

On behalf of the team and everyone who has contributed, I’m delighted to announce the first milestone of the `2025.0.0` release train. If you are confused how on earth [a milestone of `2025.0.0`](https://spring.io/blog/2025/01/24/spring-data-2025) can ship after an earlier `2025.1.0` milestone, rest assured, we are too, at times. Calver versioning is based off the actual release date. `2025.0` will go GA in May while `2025.1` is planned for November. Since we started working on the major release `2025.1` last year already, we were able to ship a first milestone in January. Hopefully, this explanation helps to clear up any confusion.

This milestone ships with enhancements, bugfixes, and dependency upgrades. This is a curated list of new and noteworthy changes in this release:

-   `Vector` domain type to represent AI embeddings using floating points and store-native Vector types.
-   MongoDB Vector Search support through [MongoDB Atlas](https://www.mongodb.com/docs/atlas/atlas-vector-search/vector-search-overview/).
-   Cassandra 5 support and [Cassandra Vector Search support](https://cassandra.apache.org/doc/latest/cassandra/getting-started/vector-search-quickstart.html).
-   `Not`, `NotIn`, and `NotLike` keyword support for Map-based KeyValue repositories.

You can find more details in our [Release Notes](https://github.com/spring-projects/spring-data-commons/wiki/Spring-Data-2025.0-Release-Notes).

The upcoming Spring Boot milestone will pick up the release for your convenience by next week.
