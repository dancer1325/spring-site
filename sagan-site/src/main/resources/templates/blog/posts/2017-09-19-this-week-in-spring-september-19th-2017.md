# This Week in Spring - September 19th, 2017

_Engineering | Josh Long |  September 19, 2017 | 2 Comments_

Hi Spring fans! Welcome to another installment of *This Week in Spring*! This week I've been visiting the Spring and Cloud Foundry teams at Microsoft (this time, in Redmond, WA) and then it's off to San Francisco, CA and San Antonio, TX to visit some customers. So, with that, let's get to it!

-   I loved this Pivotal Engineering post on how [we transitioned Pivotal Web Services, our hosted, multitenant installation of Cloud Foundry, to CredHub](http://engineering.pivotal.io/post/credhub-on-pws/). CredHub is designed to store passwords, keys, certificates, and other sensitive information for a BOSH-managed environment.
-   Spring Boot ninja Brian Clozel just announced [Spring Boot 1.5.7](https://spring.io/blog/2017/09/12/spring-boot-1-5-7-available-now) which includes scores of fixes, dependency updates, and improvements.
-   Spring Integration lead and messaging ninja Gary Russell just [announced Spring AMQP 2.0 RC1](https://spring.io/blog/2017/09/12/spring-amqp-2-0-release-candidate-1-7-4-and-1-6-11-are-available). The RC1 adds some minor improvements since the last milestone. Gary also announced that maintenance releases 1.7.4 and 1.6.11 are also available now.
-   Not one to sit on his laurels, Gary *also* announced [Spring for Apache Kafka 2.0 and 1.3 release candidates](https://spring.io/blog/2017/09/12/spring-for-apache-kafka-2-0-and-1-3-release-candidates-available). Both releases support the Kafka 0.11.x.x client, while still supporting Spring Framework 4.3. The new release includes support for Spring Framework’s transaction synchronization, a `KafkaAdmin` object, and correct handling of `AckMode.RECORD`. Also, these releases includes several bug fixes.
-   Spring Integration contributor Artem Bilan just [announced Spring Integration 5. M7 and 4.3.12](https://spring.io/blog/2017/09/14/spring-integration-5-0-milestone-7-and-4-3-12-available). The new release extracts out the reactive Spring Webflux adapter into a separate module, `spring-integration-webflux`. It also supports encoding message headers in the payload of a message for protocols - like AWS Kinesis, early versions of Apache Kafka, and TCP/IP - that don't support the notion of headers natively.
-   Spring Integration lead [Gary Russell just announcced Spring Cloud Stream Ditmars/1.3 RC1](https://spring.io/blog/2017/09/14/spring-cloud-stream-ditmars-1-3-release-candidate-announcement). The new supports working with Kafka Streams `KStream` references as inputs and outputs to `@StreamListener` methods. Put another way: you can use Spring Cloud Stream to connect your application, declare all the bindings and so on, just as normal, and then write messaging code that taps Kafka Streams for interactive queries.
-   Spring Boot committer Stéphane Nicoll just [announced Spring Boot 2.0.0.M4](https://spring.io/blog/2017/09/15/spring-boot-2-0-0-m4-available-now). This release is the first to include the new web-runtime agnostic Actuator endpoints as well a raft of changes to simplify security configuration.
-   Spring Session lead [Rob Winch just announced Spring Session 2.0.0.M4](https://spring.io/blog/2017/09/15/spring-session-2-0-0-m4). This release eases configuration for Spring WebFlux-based applications. It also supports a strategy for session ID resolution.
-   Spring ninja Greg Turnquist just [announced Spring Session MongoDB 2.0.0.M3](https://spring.io/blog/2017/09/15/spring-session-mongodb-2-0-0-m3-released). The new release builds on Spring Session 2.0.0.M4, Spring Data Kay RC3, Reactor Bismuth-M4, and Spring Framework 5.0.0.RC4. If you're using Spring Boot 2.0's `spring-boot-starter-data-mongodb-reactive`, then all you'll need is `@EnableMongoWebSession`!
-   Spring Security lead Rob Winch just [announced Spring Security 5.0.0.M4](https://spring.io/blog/2017/09/15/spring-security-5-0-0-m4-released). This release is full of great new features including OAUth 2 / OpenID Connect support and a reactive integration for Spring WebFlux.
-   Spring web guru Rossen Stoyanchev just [announced Spring Web Flow 2.4.6](https://spring.io/blog/2017/09/15/spring-web-flow-2-4-6-released). The release includes a security fix that affects default bindings.
-   Spring Cloud Data Flow contributor Eric Bottard [just announced Spring Shell 2.0.M1](https://spring.io/blog/2017/09/18/introducing-spring-shell-2-0m1)! The Spring Shell project surfaces a *command* component model to support developing custom shells. Spring Shell is the project that underpins, for example, the Spring Cloud Data Flow shell. This new release has a *lot* to offer including positional parameters, use of the Spring conversion API, JLine 3 support, integration with the bean validation API, multi-line commands, and much more.
-   Simon Baslé just [announced Reactor 3.1 RC1](https://spring.io/blog/2017/09/18/announcing-first-release-candidate-of-reactor-core-3-1). This is a huge step on the road to Spring Framework 5.0. Users are *strongly* encouraged to grab the bits and try things out ASAP!
-   Spring Cloud Data Flow contributor Glenn Renfro [announced the 1.3.0.M2 release](https://spring.io/blog/2017/09/18/spring-cloud-data-flow-1-3-0-m2-released), packed with great new features. It includes an updated Dashboard/Flo integration supporting batch, task and streaming operations; fan-in and fan-out support; application registry and Maven update policies; security updates; direct named channels; shell improvements and much more.
-   Spring Boot ninja Madhura Bhave looks at the ways that [Spring Boot 2.0.0.M4 improves its integration with Spring Security](https://spring.io/blog/2017/09/15/security-changes-in-spring-boot-2-0-m4); this is definitely worth a read!
-   Tom Hombergs does a good [job looking at messaging for microservices with Spring Boot and RabbitMQ](https://reflectoring.io/event-messaging-with-spring-boot-and-rabbitmq/).
-   Another friendly reminder that I'll be [speaking at the Reactive Summit](https://www.youtube.com/watch?v=GzebF1iKZPU) event on Reactive Spring and I hope to see you there!
-   A friendly reminder that I'll be speaking at [Kotlin Conf](https://kotlinconf.com/speakers/#speaker=josh-long) on Spring Boot and Kotlin in December and I'd love to see you there.
-   Damith Ganegoda does a nice job introducing [Spring Boot and Angular 4](http://mydevgeek.com/angular-4-crud-application-with-spring-boot-rest-service-part-1/) in this blog post.
-   The Ordina JWorks blog [has a great look at the new Micrometer monitoring abstraction](http://ordina-jworks.github.io/microservices/2017/09/17/monitoring-your-microservices-with-micrometer.html) (which is also integrated in Spring Boot 2.0.0.M4).
-   Atomist's Clay McCoy looks at how Atomist can [help you catch breaking REST API changes in your Spring Boot application](https://the-composition.com/change-is-the-only-constant-in-a-rest-api-318baf92301f).
-   I liked this post on ZDNet [on Greenplum, big data's best kept secret](http://www.zdnet.com/google-amp/article/pivotal-greenplum-is-alive-and-kicking/)
-   Speaking of Spring Boot committer and all around amazing human Madhura BHave, she *also* put together a nice look [at what's coming in Spring Boot 2.0 for her JavaZone 2017 presentation](https://speakerdeck.com/mbhave/a-sneak-peek-into-spring-boot-2-dot-0).
-   I like Dan's post that introduces the concepts of hypermedia and HATEOAS, in particular, and does so with [some sample code based on Spring Boot and Spring HATEOAS](https://lankydanblog.com/2017/09/10/applying-hateoas-to-a-rest-api-with-spring-boot/amp/).
-   Facebook and Github have just announced [Atom IDE](https://github.com/blog/2430-introducing-atom-ide) - a new IDE that, apparently, supports Java. I haven't had a chance to try it yet but it looks interesting. I wonder if it'll have Maven or Gradle support..
