# Spring Framework 6.0 M1 released

_Releases | Juergen Hoeller |  December 16, 2021 | 8 Comments_

Dear Spring community,

It is my pleasure to announce that the first Spring Framework 6.0 milestone release is available from [htts://repo.spring.io/milestone](https://repo.spring.io/) now. This initial milestone covers our baseline upgrade efforts, in particular requiring JDK 17+ and migrating to the Jakarta EE 9 APIs; see my [recent baseline blog post](https://spring.io/blog/2021/09/02/a-java-17-and-jakarta-ee-9-baseline-for-spring-framework-6) for the rationale. At the same time, it removes many long-deprecated classes, including several support packages for outdated third-party infrastructure.

For current upgrade notes, please refer to our [Upgrading to Spring Framework 6.0 page](https://github.com/spring-projects/spring-framework/wiki/Upgrading-to-Spring-Framework-6.x) which we will keep updating. Follow our main branch on [GitHub](https://github.com/spring-projects/spring-framework/) for the latest changes, as we prepare for 6.0 M2 and the corresponding Spring Boot 3.0 M1 release in January. At that point, you will also be able to consume Spring Framework 6.0 through [https://start.spring.io/](https://start.spring.io/). For the time being, feel free to grab 6.0 M1 from [https://repo.spring.io/](https://repo.spring.io/) and let us know how it goes.

Cheers, Juergen
