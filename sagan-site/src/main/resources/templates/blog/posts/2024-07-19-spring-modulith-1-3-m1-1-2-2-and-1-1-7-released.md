# Spring Modulith 1.3 M1, 1.2.2, and 1.1.7 released

_Releases | Oliver Drotbohm |  July 19, 2024 | 1 Comment_

I am excited to announce the releases of Spring Modulith 1.3 M1, 1.2.2, and 1.1.7. The former ships with the following new features:

-   Upgrade to Spring Boot 3.4 M1 and Spring Framework 6.2 M6 – [GH-737](https://github.com/spring-projects/spring-modulith/issues/737)
-   Improvements in the documentation creation – [GH-644](https://github.com/spring-projects/spring-modulith/issues/644), [GH-638](https://github.com/spring-projects/spring-modulith/issues/638)
-   `@ApplicationModuleTest` can now be used outside a module folder – [GH-466](https://github.com/spring-projects/spring-modulith/issues/466)
-   A configuration property to define the schema of the Event Publication Registry tables for JDBC – [GH-685](https://github.com/spring-projects/spring-modulith/issues/685)

These features are surrounded by a few bug fixes that have all been backported to the 1.2 and 1.1 generations of Spring Modulith. Here you can find the full change logs for the releases [1.3 M1](https://github.com/spring-projects/spring-modulith/releases/tag/1.3.0-M1), [1.2.2](https://github.com/spring-projects/spring-modulith/releases/tag/1.2.2), and [1.1.7](https://github.com/spring-projects/spring-modulith/releases/tag/1.1.7).

We appreciate any kind of feedback, in particular regarding the milestone release, in the GitHub [discussions](https://github.com/spring-projects/spring-modulith/discussions).
