# This Week in Spring - October 4th, 2016

_Engineering | Josh Long |  October 05, 2016 | 0 Comments_

Welcome to another installment of *This Week in Spring*! We've got a lot to look at so let's get to it!

-   Spring Security lead Rob Winch [just released Spring Security LDAP 2.2RC1](https://spring.io/blog/2016/10/04/spring-ldap-2-2-rc1)
-   Spring Integration lead Gary Russell [just announced Spring Integration and Spring AMQP release candidates](https://spring.io/blog/2016/10/03/spring-integration-amqp-maintenance-releases-available); get the bits now!
-   Spring Data ninja Thomas Risberg just [released Spring Cloud Data Flow for Kubernetes 1.1.M1](https://spring.io/blog/2016/10/03/spring-cloud-data-flow-for-kubernetes-1-1-m1-and-1-0-1-ga-released)
-   Spring Data [lead Oliver Gierke announced Spring Data Hopper SR4 and Spring Data Gosling SR6](https://spring.io/blog/2016/10/03/spring-data-hopper-sr4-and-gosling-sr6-released)
-   Spring Statemachine lead [Janne Valkealahti just announced Spring Statemachine 1.2.0M1](https://spring.io/blog/2016/09/29/spring-statemachine-1-2-0-m1-released)
-   I liked this GR8Conf talk by Jenn Strater on a test-driven approach to documenting REST APIs [with Spring REST Docs](https://www.youtube.com/watch?v=-HocWtMO3FY)
-   This German language news roundup ever so briefly [looks at Spring 5 and Rod Johnson's new endeavor, Atomist](https://jaxenter.de/spring5-eclipse-unide-atomist-kubernetes-rust-machine-learning-47485)
-   [JAXenter interviewed me, as well](https://jaxenter.com/most-important-benefit-microservices-is-agility-129472.html)
-   check out [what's coming in Netflix Zuul's future](https://www.infoq.com/news/2016/10/netflix-zuul-asynch-nonblocking) in this nice InfoQ post
-   EMC's Paul Warren has some interesting projects you might dig: [check out Ginkgo4j do a BDD and Ginkgo-style test using Java](https://paulcwarren.github.io/ginkgo4j/) and check out [his Spring Data extensions to support arbitrary blob storage](https://paulcwarren.github.io/spring-content/)

Also, I wanted to note the loss today [of Pieter Hintjens](http://hintjens.com), who - among his many contributions to the world - was the original designer of what would become the AMQP protocol that powers RabbitMQ. Rest in peace, Pieter.
