# This Week in Spring - January 17th, 2017

_Engineering | Josh Long |  January 17, 2017 | 1 Comment_

Welcome to another installment of *This Week in Spring*! We've got a lot to get into this week - even *more* so than usual! So, let's get into it!

This week there are a *lot* of great SpringOne Platform 2016 replays available online thanks to our very own [Pieter Humphrey](http://twitter.com/PieterHumphrey)!

-   Don't miss Stéphane Maldini's SpringOne Platform (S1P) 2016 talk introducing [Reactor 3.0](https://spring.io/blog/2017/01/16/springone-platform-2016-replay-reactor-3-0-a-jvm-foundation-for-java-8-and-reactive-streams)
-   or this S1P 2016 talk by Sebastien Deleuze and Stephane Maldini [on reactive programming options for the JVM](https://spring.io/blog/2017/01/16/springone-platform-2016-replay-a-lite-rx-api-for-the-jvm) (introducing Reactor and RxJava, among others)
-   Yahoo! [Marcin just tweeted](https://twitter.com/mgrzejszczak/status/819955779000213505) some exciting news: [Pact support has been added to Spring Cloud Contract](https://cloud.spring.io/spring-cloud-contract/spring-cloud-contract.html#_pact_converter)
-   Noriaki Tatsumi ( from Capital One) gave a nice S1P 2016 talk on [operating a high velocity organization with Spring Cloud](https://spring.io/blog/2017/01/16/springone-platform-2016-replay-operating-a-high-velocity-large-organization-with-spring-cloud-microservices)
-   Google's Ray Tsang and Bret McGowen put together a nice [talk on using gRPC for Spring Developers](https://spring.io/blog/2017/01/16/springone-platform-2016-replay-grpc-101-for-spring-developers)
-   in this S1P 2016 talk, Ben Hale, Paul Harris and Stephane Maldini introduce some [of the nuance of building reactive APIs](https://spring.io/blog/2017/01/16/springone-platform-2016-replay-designing-implementing-and-using-reactive-apis)
-   Ryan Baxter has just announced [Spring Cloud Camden.SR4](https://spring.io/blog/2017/01/12/spring-cloud-camden-sr4-is-available)
-   Glenn Renfro just [announced Spring Cloud Task 1.1.2.RELEASE](https://spring.io/blog/2017/01/10/spring-cloud-task-1-1-2-release-is-now-available)
-   speaking of Spring Cloud Contract, I really dig this post by epages' Mathias on building [and testing event-driven microservices with Spring Cloud Contract](https://developer.epages.com/blog/2017/01/17/how-to-test-eventbased-services-using-contracts.html)
-   I *loved* our very own Toshiaki Maki's exhaustive repository demonstrating how to [use Spring Boot with a larger number of different SQL persistence technologies](https://github.com/making/spring-boot-db-samples). *Great* resource! Bookmark this!
-   and speaking of persistence options, I really enjoyed community hero [Michael Simmons' talk on JOOQ and Spring Boot](https://www.youtube.com/shared?ci=N3HpOiGbM60)
-   Hortonworks's Tim Spann got a few great posts on [using Spring Boot with Apache Phoenix or Apache Hive](https://community.hortonworks.com/topics/microservice.html)
-   Martin Westergaard Lassen has put together a nice post on his top favorite seven (ahem! six.. :D) [features of Spring Boot](http://container-solutions.com/my-top-7-favourite-features-of-spring-boot/)
-   I really liked this post by Apigee's Alan Ho on the reasons [that organizations diffuse monolithic time-bombs](http://sdtimes.com/guest-view-microservices-death-diffusing-monolith-time-bomb/): "If the monolithic stack doesn’t go, the good developers will"
-   Yahoo! just open-sourced their [continuous delivery platform called Screwdriver](http://sdtimes.com/yahoo-open-sources-continuous-delivery-tool-screwdriver/). This has little or nothing to do with Spring or Pivotal, directly, yet, but it's yet another contender in the ever expanding ecosystem of continuous delivery tools, along with Pivotal's Concourse, Netlfix Spinnaker (to which Pivotal, Google, Microsoft and Netflix all commit and contribute), Jenkins and so many more. Remember, at the end of the day, the most important metric for organizational agility isn't cloud computing or microservices, it's the ability to continuously (quickly and safely) deliver value to the customer.
-   This is super cool! Now you [can use Apache Camel with Spring Cloud Stream](https://blog.switchbit.io/camel-spring-cloud-stream/)
-   this is a topic near and dear to my heart: using [a Spring Boot Slack bot to notify people of the availability of coffee](https://github.com/aglassman/coffee-bot)
-   Hurray! [Cloud Foundry surgical routing is now supported in Spring Cloud Ribbon-based load-balancing](https://github.com/spring-cloud/spring-cloud-commons/issues/162)
-   want to consume of the Spring Cloud Services from .NET applications? [Check out this post on Steeltoe](https://www.altoros.com/blog/enabling-dotnet-core-microservices-with-steeltoe-and-pivotal-cloud-foundry/)
-   Evan Brown put [together a succinct blog](https://cloudplatform.googleblog.com/2017/01/partnering-on-open-source-Google-and-Pivotal-engineers-talk-Cloud-Foundry-on-GCP.html) introducing a replay of a talk on how to deploy Pivotal Cloud Foundry on Google Cloud Platform. [Check out this talk on BOSH and Cloud Foundry Basics for Google Cloud Platform, pt. 1](https://www.youtube.com/watch?v=TBsc7kiog5Q)
-   want to learn how to use Spring Cloud Security (and its OAuth support for clients) with [RedHat's Keycloak? Check this blog out by RedHat's Kamesh out](https://developers.redhat.com/blog/2017/01/05/spring-boot-and-oauth2-with-keycloak/)
