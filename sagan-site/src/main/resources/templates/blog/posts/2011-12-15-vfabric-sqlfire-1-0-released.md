# vFabric SQLFire 1.0 Released

_Releases | Adam Fitzgerald |  December 15, 2011 | 0 Comments_

[![VMware vFabric SQLFire](http://www.springsource.org/files/SQLFireLogo_128x128.png)](http://www.vmware.com/go/sqlfire)

VMware's [vFabric SQLFire](http://www.vmware.com/go/sqlfire) became GA today. It is a memory-optimized distributed SQL database delivering dynamic scalability and high performance for data-intensive modern applications. SQLFire provides developers with a simple SQL interface and Spring developers can connect their application easily using plain JDBC.

-   The [SQLFire blog](http://blogs.vmware.com/sqlfire/) by Jags Ramnaryan provides an excellent introduction to the problems that this product helps to solve.
-   Jags also provides a great video [Introduction to SQLFire](http://www.youtube.com/watch?v=d7Sq0CoJS4M) on YouTube.
-   There is a great [step by step tutorial](https://www.vmware.com/go/sqlfire-tutorial) to get you started with SQLFire.
-   Interested developers can always ask questions in the [SQLFire community](http://communities.vmware.com/community/vmtn/appplatform/vfabric_sqlfire) discussion forum.

So if you are looking for a high performance, dynamically scaling data solution that has simple SQL access, be sure to [download SQLFire](http://www.vmware.com/tryvmware/?p=sqlfire&lp=default) and give it a try.
