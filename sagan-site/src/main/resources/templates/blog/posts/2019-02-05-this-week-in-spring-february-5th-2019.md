# This Week in Spring - February 5th, 2019

_Engineering | Josh Long |  February 05, 2019 | 0 Comments_

Hi Spring fans! Welcome to another installment of *This Week in Spring*! First of all, [Happy Lunar New Year](https://www.bbc.com/news/world-asia-47121421), if you celebrate! (More than a billion people around the world do celebrate...)

Today I'm going to be giving a 4-hour presentation, [*Reactive Spring*](https://www.safaribooksonline.com/live-training/courses/reactive-spring-boot/0636920240426/), for Safari members.

So, without further ado, let's get to it!

-   [Spring Session for Apache Geode/Pivotal GemFire 2.0.8.RELEASE available!](https://spring.io/blog/2019/02/04/spring-session-for-apache-geode-pivotal-gemfire-2-0-8-release-available)
-   Last week in *A Bootiful Podcast*, I interviewed Spring messaging and integration maestro, and one of my oldest friends on the Spring team, [Oleg Zhurakousky](https://spring.io/blog/2019/02/01/a-bootiful-podcast-oleg-zhurakousky).
-   Hi Spring fans! In last week's installment of *Spring Tips*, [*The Reactor Context*](https://spring.io/blog/2019/01/30/spring-tips-the-reactor-context), I look at how you can perpetuate state across thread execution jumps.
-   If you're using older versions of Cloud Foundry then you might be using [a vulnerable version of Java](https://www.cloudfoundry.org/vulnerable-jdk-8u182/)
-   I really liked this response from Kenny Bastani on [eventual consistency](https://twitter.com/springcentral/status/1092022447606362113)
-   This is an interesting post on using [Spring Boot with Vault on Kubernetes](https://banzaicloud.com/blog/vault-java-spotguide/)
-   I love this - a look at using [Neo4j and Spring Data Neo4j](https://twitter.com/springcentral/status/1091362057629417474) with the epic Mark Heckler and Jennifer Reif, among others
-   I loved this look at Spring Data Redis and [high-availability with Sentinel](https://twitter.com/springcentral/status/1090682564799549442)
-   Check out this replay of the [Securing Microservices with Spring and Pivotal Cloud Foundry](https://www.brighttalk.com/webcast/14893/345709) webinar
-   Hey check this out - it's a third-party thing you can use to get lightweight administration of Spring Batch jobs, [Spring Batch Lightmin](https://github.com/tuxdevelop/spring-batch-lightmin)
-   This is interesting - it's a program from `pacphi` called `cf-butler` that cleans stale apps and services on [a Cloud Foundry foundation](https://github.com/pacphi/cf-butler). It can be configured to report on and remove orphaned services and stopped app instances older than a configurable duration.
