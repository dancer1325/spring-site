# This Week in Spring - October 18, 2016

_Engineering | Josh Long |  October 18, 2016 | 0 Comments_

Welcome to another installment of *This Week in Spring*! It's been another crazy wonderful week. This week I'm in London and Cambridge, UK visiting with customers and speaking (in particular, at the O'Reilly [**Software Architecture Conference**](http://conferences.oreilly.com/software-architecture/engineering-business-eu)) If you're about be sure to say hi! Now then, let's get to it!

-   The Spring Cloud Services Team, led by [Chris Sterling](https://www.twitter.com/csterwa) has just finished making Eureka Service Discovery work even better on Pivotal Cloud Foundry. The [new 1.2 release](https://blog.pivotal.io/pivotal-cloud-foundry/products/new-in-spring-cloud-services-1-2-multi-site-service-discovery) allows peer replication across installations and organizations, enabling multi-site replication.
-   Spring Cloud Data Flow 1.1.M2 just released today, picking up Boot 1.4.1 and Spring Cloud Camden releases. Check out all the extra goodies in [Mark Pollack's blog post](https://spring.io/blog/2016/10/18/spring-cloud-data-flow-1-1-m2-released).
-   Learn how to build data-centric microservices [with Spring Cloud Data Flow with this incredible webinar](http://spring.io/blog/2016/10/17/webinar-replay-data-microservices-in-the-cloud)
-   Spring Integration ninja Artem Bilan has just [announced the Java DSL for Spring Integration 1.2](http://spring.io/blog/2016/10/14/java-dsl-for-spring-integration-1-2-release-is-available)
-   Last week I put together a quick look - another installment in the *Spring Tips* series! - at how [to build application integration flows with Spring Integration](http://spring.io/blog/2016/10/13/spring-tips-a-gentle-introduction-to-spring-integration) (and the aforementioned Java DSL!). Enjoy.
-   I really liked this InfoQ article Edith Harbaugh on [the perils and potential of feature flags in a system](https://www.infoq.com/articles/feature-flags-gone-wrong)
-   Are you on macOS Sierra and having trouble with scrolling in IntelliJ IDEA? [Check out this *super useful* tip](https://youtrack.jetbrains.com/oauth?state=%2FissueMobile%2FIDEA-158500) for a workaround!
-   I liked this article on [design principles when building APIs with Java 8](https://dzone.com/articles/the-java-8-api-design-principles)
-   Want to see what the future of reactive Spring Security looks like? [Follow along this repository as the Spring Security team starts work](https://github.com/spring-projects/spring-security-reactive) but, *please*, mind the dust!
-   Volkswagen, yet another car company that *gets it*, [has chosen Pivotal Cloud Foundry to reduce cycle time](http://www.computerworlduk.com/cloud-computing/volkswagen-continues-open-source-cloud-push-with-cloud-foundry-paas-3646928/) from the order of years to down to a week (and I wouldn't be surprised if they could get that down further!)
-   our pal, Michael Simmons, has put together a great [post on referencing `@MockBean`s by name in a Spring test](http://info.michael-simons.eu/2016/10/13/spring-boot-referencing-mockbeans-by-name/)
-   Are you in Japan? [Don't miss this Spring Days event!](http://springday2016.springframework.jp/)
-   Spring Data lead Oliver Gierke put together a nice post on [the tradeoffs of building an evolvable system](http://olivergierke.de/2016/10/evolving-distributed-systems/).
-   Jonas Hecht has posted yet another installment in his series looking at how to work with CXF and Spring, this time debuting a [good-looking Spring Boot auto-configuration and starter that simplifies things *considerably*](https://blog.codecentric.de/en/2016/10/spring-boot-apache-cxf-spring-boot-starter/)
-   ZeroTurnaround have [posted a remix of their data from a recent survey](https://zeroturnaround.com/rebellabs/spring-vs-java-ee-survey-results/) of the Java community, this time breaking down use of different technologies - Spring and Java EE - from among survey participants.
-   TheNewStack is back at it again, this time with a post called [*Cloud Foundry Remains King over a Changing Platform Landscape*](http://thenewstack.io/embracing-pragmatism-software-development/). Interesting analysis!
