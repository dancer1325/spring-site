# This Week in Spring - June 6th, 2017

_Engineering | Josh Long |  June 06, 2017 | 0 Comments_

Welcome to another installment of *This Week in Spring*! This week I'm in Malaysia and then Singapore where I'll spend time with customers in the financial services world and speak at the [Singapore Spring User Group](https://www.meetup.com/singasug/) on June 8th. If you're around, come out or [say hi](http://twitter.com/starbuxman)!

-   Spring IO Platform lead and Spring Boot ninja Andy Wilkinson just announced the [Gradle Dependency management plugin 1.0.3](http://spring.io/blog/2017/06/05/dependency-management-plugin-1-0-3-release)
-   Spring web guru has just [announced Spring Web Flow 2.4.5](http://spring.io/blog/2017/05/31/spring-web-flow-2-4-5-released)
-   the Baeldung blog has a nice look [at Spring Boot and Kotlin](http://www.baeldung.com/spring-boot-kotlin)
-   Ranga Karanam put together a nice look at building [web applications with Spring Boot](https://dzone.com/articles/creating-a-web-application-with-spring-boot)
-   I liked this blog offering tips and tricks for using the compile-time annotation [processor Lombok in production application](https://dzone.com/articles/5-tips-for-using-lombok-in-production). Lombok is not a Spring project, obviously, but it's easy to use with Spring and it's available on the Spring Initializr \[as a dependency\].
-   I reallly enjoyed some of the replies in the Quora post [*When would you choose Couchbase vs MongoDB vs Redis*](https://www.quora.com/NoSQL-When-would-you-choose-CouchBase-vs-MongoDB-vs-Redis)
-   Cloud Foundry is dropping Consul from its internal implementation. This design doc / post-mortem looks at why, in this particular case, [it makes sense for Cloud Founddry to move Consul out of the stack](https://docs.google.com/document/d/1qdLNIWQQzluXw5rnc39raAYOnnSdDUjhUOrovUE0NJI/edit#heading=h.f54rg2dpu35y)
-   The Baeldung blog also updated their [Java 9 Reactive Streams content, which is as-always a worthy read](http://www.baeldung.com/java-9-reactive-streams)
-   I liked this talk by Martin Fowler [on the different uses of messaging](https://www.youtube.com/watch?v=STKCRSUsyP0)
-   Pivotal's very own Matt Stine put together a nice read, [*What's your Decomposition Strategy?*](https://builttoadapt.io/whats-your-decomposition-strategy-e19b8e72ac8f)
-   interesting analysis from Bob Evans at Forbes: [*Sorry Amazon, but Microsoft is the World's #1 Cloud Vendor*](https://www.forbes.com/sites/bobevans1/2017/06/01/sorry-amazon-but-microsoft-is-the-worlds-1-cloud-vendor-heres-why-cloud-wars/#1a83af887928)
-   I loved [this Dreamworks case-study](https://builttoadapt.io/how-to-train-your-microservices-ef2d912109d9)
-   Coda Hale (who, among *many* things, created the DropWizard Metrics library) put together a great [read introducing Little's Law](https://codahale.com/usl4j-and-you/) - an approach to modeling system performance - and his new library, [usl4j](https://github.com/codahale/usl4j).
-   Pivotal's own Joshua McKenty put together a nice piece on how [to diversify a cloud portfolio](https://builttoadapt.io/diversify-your-cloud-portfolio-or-bank-on-failure-dabaead526a1).
