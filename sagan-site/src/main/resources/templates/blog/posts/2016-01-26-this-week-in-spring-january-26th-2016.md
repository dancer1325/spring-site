# This Week in Spring - January 26th, 2016

_Engineering | Josh Long |  January 26, 2016 | 0 Comments_

Welcome to another installation of *This Week in Spring*! Here we are, already staring at the second month of the year and, *boy*!, are things getting exciting! I'm about to start hitting the road again and [I'd](http://twitter.com/starbuxman) love [to talk to you](mailto:josh@joshlong.com) and your organization if you're near any of the following places and events in February:

-   [Microchg Conf](http://microxchg.io/2016/index.html)
-   [JFokus](http://www.jfokus.se/jfokus/)
-   [Devnexus](https://www.devnexus.com/s/index)
-   Hatford, Connecticut (around 17/2/2016)
-   [Confoo](https://confoo.ca/en)
-   Shanghai (around 26/2/2016)

Anyway, without further ado, let's get into it!

-   our own Paul Chapman chimes in on the [updated courseware available for developers who want to learn about Spring](https://spring.io/blog/2016/01/21/pivotal-academy-launches-updated-spring-classes)
-   there are (finally!) new guides available on using **Spring Cloud**: [routing and filtering using the Netflix Zuul microproxy library and Spring Cloud](https://spring.io/guides/gs/routing-and-filtering/), [centralized configuration with the Spring Cloud Config Server](https://spring.io/guides/gs/centralized-configuration/), the [circuit breaker design pattern](https://spring.io/guides/gs/circuit-breaker/), and service registration and discovery [using Spring Cloud's `DiscoveryClient` abstraction and Spring Cloud's integration of Netflix Eureka](https://spring.io/guides/gs/service-registration-and-discovery/). Stay tuned, we've got more in the pipeline!
-   Spring Boot lead Phil Webb just announced [the release of 1.3.2](https://spring.io/blog/2016/01/22/spring-boot-1-3-2-released), which includes updates to dependencies and a few bug fixes
-   check out Jetbrain's Yann Cébron and our very own Stéphane Nicoll's [epic SpringOne2GX 2015 presentation on how Spring Boot lends itself to first-class tooling](http://www.infoq.com/presentations/spring-boot-intellij)
-   our pal Baeldung is back at it again, [this time with a video on monitoring Spring Boot applications](https://www.youtube.com/watch?v=71UFLAVXgq0)
-   [http://tomaszdziurko.pl/2016/01/dockerizing-spring-boot-application/](http://tomaszdziurko.pl/2016/01/dockerizing-spring-boot-application/)
-   Neo4j has a nice transcription [of our GraphConnect 2015 talk](http://neo4j.com/blog/polyglot-persistence-microservices-spring-cloud-neo4j/)
-   this third entry in a Spanish-language series [on building microservices with Spring Boot and Spring Cloud looks pretty good!](https://www.paradigmadigital.com/dev/integrar-microservicios-spring-cloud-34/)
-   the talk I gave on building cloud native Java applications [using Spring Boot and Spring Cloud at QCon Rio de Janeiro](http://www.infoq.com/br/presentations/the-cloud-native-application-microservices-with-spring-boot-and-spring-cloud)
-   I really dug our pal Nicolas Frankel's post [on using Spring Boot and Vaadin together with the Kotlin programming language](https://blog.frankel.ch/playing-with-spring-boot-vaadin-and-kotlin). Kotlin's a very nice language and this post covers some issues you'll encounter in your first steps using Spring and Kotlin together. Very handy!
-   Groovy is *also* a very nice programming language and if you want further proof then you need look [no further than this post which demonstrates standing up a Spring Cloud Config Server in almost no code at all!](https://dzone.com/articles/grails-goodness-using-spring-cloud-config-server)
