# This Week in Spring - September 5th, 2017

_Engineering | Josh Long |  September 04, 2017 | 1 Comment_

Hi Spring fans and welcome to another installment of *This Week in Spring*! This week I'm in Hong Kong for the amazing [YOW! Hong Kong](http://www.yowconference.hk/) event and then it's off to Singapore for the [YOW! Singapore](http://www.yowconference.sg) event. If you're in either region, these conferences are world-class events from Australia and they're new in the region so I hope you'll give them a shot.

-   Check out this first of many introductions to various aspects of Spring Cloud Data Flow. This first introduction, by Spring Cloud Data Flow ninja Eric Bottard, [looks at the Spring Cloud Data Flow shell](https://spring.io/blog/2017/08/29/spring-cloud-data-flow-tips-tricks-interacting-with-the-shell) itself.
-   Spring Cloud contributor [Ryan Baxter just announced Spring Cloud Edgeware M1](https://spring.io/blog/2017/08/29/spring-cloud-edgware-m1-is-now-available). The new release includes updates to the various projects as well as the new Spring Cloud Gateway project. This version is based on Spring Boot 1.5.x.
-   Spring Cloud co-founder Spencer Gibb [just announced Spring Cloud Finchely M2](https://spring.io/blog/2017/08/30/spring-cloud-finchley-m2-is-available). This new release is based on Spring Boot 2.0.x and so has a Java 8-baseline. It also supports a lot more cool features in Spring Cloud Gateway. A common theme of this release, in Spring Cloud Gateway and other releases, is reactive support.
-   Spring Cloud Data Flow ninja Thomas Risberg [introduces the Helm installer for Spring Cloud Data Flow for Kubernetes in this post](https://spring.io/blog/2017/08/31/simple-installation-of-data-flow-for-kubernetes-with-helm). Helm is a package manager for Kubernetes.
-   This Pivotal and Apigee whitepaper on [application modernization was a pretty good read](https://content.pivotal.io/apigee/application-modernization?_lrsc=9d76ba9e-eb43-42fb-b85d-9e5b48373ac4)
-   This is an oldie-but-a-goodie from earlier this year by Matt McCandless called [*Spring Boot: The Right Boot for You*](https://dzone.com/articles/spring-boot-the-right-boot-for-you-1)
-   this post on [the Quartz Scheduler with Spring Boot](http://www.opencodez.com/java/quartz-scheduler-with-spring-boot.htm) was a blast-from-the-past sort of article. I dig it.
-   It's looking like there will be [a 6 month cadence for new Java releases](https://jcp.org/aboutJava/communityprocess/ec-public/materials/2017-08-15/JCP-EC-Minutes-August-2017.html) after Java 9 ships. I personally welcome this idea.
-   I really liked this Russian-language [post introducing application development with Spring Boot](https://alexkosarev.name/2017/08/08/application-development-with-spring-part-1/).
-   Last week Pivotal, in conjunction with [Google](https://www.blog.google/topics/google-cloud/vmware-and-pivotal-launch-new-hybrid-kubernetes-solution-optimized-gcp/) and VMWare, announced a very important next step for the Pivotal Cloud Foundry ecosystem - *Pivotal Container Service* - a Kubernetes-service managed by BOSH. Get the skinny on how [we got here from this TechCrunch piece](https://techcrunch.com/2017/08/29/seven-moves-that-led-to-the-vmware-pivotal-google-partnership/amp/)
-   I loved this Auth0 piece on integrating [Spring Boot with a SQL database (PostgreSQL), JPA and Liquibase](https://auth0.com/blog/integrating-spring-data-jpa-postgresql-liquibase/).
-   this post from JHipster on [Doing API-First development](http://www.jhipster.tech/doing-api-first-development/) was interesting.
-   Spring Boot 2.0, with a reactive Spring Framework 5 and Java 8 baseline, is sure to impress. One of my favorite features in the new release is REST-stack agnostic (reactive Spring WebFlux, Spring MVC or JAX-RS) [support for the Spring Boot Actuator endpoints](https://www.infoq.com/news/2017/08/spring-boot-2-actuator-endpoints), which this blog post details.
-   Check out this excellent webinar [on why IntelliJ IDEA is the premier choice for Grails applications](https://blog.jetbrains.com/idea/2017/07/webinar-recording-why-intellij-idea-is-the-premier-ide-for-grails-3/). I love this because Grails 3 is based on Spring Boot and because Jeff Scott Brown's a great speaker and because IntelliJ make a pretty darned good IDE.
-   This isn't strictly-speaking related to Spring, but it is a worthy read. [Daniel Bryant describes a wholistic approach to testing microservices](https://specto.io/blog/2016/8/16/recipe-for-designing-building-testing-microservices/) - and all the components you might use withint them - in this post.
-   I liked this StackOverflow response from Spring Data lead Oliver Gierke: [*How do I selectively upgrade a dependency in Spring Boot?*](https://stackoverflow.com/questions/45967464/how-to-selectively-upgrade-a-dependency-in-spring-boot-sample-case-spring-dat)
-   Read this [post](http://www.automate-it.today/about-cloud-foundry-service-brokers/) if you want a quick introduction to what a Cloud Foundry service broker is.
-   This is also not strictly speaking related to Spring, but it could very well be useful to a number of Spring users building cloud-native applications: [*Cost Reduction Strategies on Java Cloud Hosting Services*](https://www.infoq.com/articles/java-cloud-cost-reduction)
