# This Week in Spring - July 26th, 2016

_Engineering | Josh Long |  July 26, 2016 | 0 Comments_

Welcome to another installation of *This Week in Spring*! This week I'm mostly in San Francisco and Saint Louis, busily preparing for the big event next week!

This is my favorite time of year! As we lead to **[SpringOne Platform](http://springoneplatform.io)**, there's so much good stuff being released that one can hardly keep up! I am *really* looking forward to this year's **[SpringOne Platform](http://springoneplatform.io)** show, coming in early August. It's an amazing time to build applications, and SpringOne Platform is in a unique position to capture the larger discussion: why do we #devops, #cloud, #agile, or #microservice? Join the discussion August 1-4, 2016 in beautiful Las Vegas and let's find out!

We've got a lot to cover and very little time in which to do it so let's get to it!

-   Spring Cloud Contract [lead Marcin Grzejszczak has just announced that M1 is now available](https://spring.io/blog/2016/07/25/spring-cloud-contract-1-0-0-m1-released). The blog features a good look at the concepts behind consumer driven contracts, too, and so is well worth a read!
-   Spring Cloud Data Flow product manager Sabby Anandan wrote up a [very nice piece for the Pivotal blog on Spring Cloud Data Flow 1.0](https://blog.pivotal.io/big-data-pivotal/products/data-goes-cloud-native-with-the-new-spring-cloud-data-flow).
-   check out this epic [presentation by Spring Cloud Data Flow leads Dr. Pollack and Mark Fisher to learn more about data-driven microservices](https://www.youtube.com/watch?v=Z6_ttC33N0U)
-   Spring Integration ninja Artem Bilan has just [announced Spring Integration AWS 1.0.0.RELEASE](https://spring.io/blog/2016/07/26/spring-integration-aws-1-0-0-ga-available-now) is now available. The final Release is a recommended update [for all those using Spring Integration AWS M1](https://spring.io/blog/2016/04/14/spring-integration-aws-1-0-0-m1-is-now-available)
-   Never one to rest on his laurels, Artem *also* [announced that Spring Integration 4.3.1, 4.2.9, 4.1.9, 4.0.9 are available](https://spring.io/blog/2016/07/26/spring-integration-4-3-1-4-2-9-4-1-9-4-0-9-are-available)
-   do you want to use Hazelcast on Cloud Foundry? [Take a look at the service broker and information on its features, here](http://blog.hazelcast.com/cloud-foundry/)
-   Spring REST Docs lead Andy Wilkinson's [just announced Spring REST Docs 1.1.1](https://spring.io/blog/2016/07/25/spring-rest-docs-1-1-1-release) which features a littany of [small improvements and bugfixes](https://github.com/spring-projects/spring-restdocs/issues?q=milestone%3A1.1.1.RELEASE).
-   [Rossen Stoyanchev just announced that Spring Web Flow 2.4.4](https://spring.io/blog/2016/07/20/spring-web-flow-2-4-4-released) is now available. This is a maintenance release that, among other things, ensures compatability with Hibernate 5.2 and fixes several small things in JSF.
-   the good Dr. Syer has published another installment in his series on building reactive applications, [this time looking at building a simple Reactive HTTP server](https://spring.io/blog/2016/07/20/notes-on-reactive-programming-part-iii-a-simple-http-server-application)
-   Krishna Srinivasan put together a nice look at how to [use some of the Spring Boot Actuator module's exposed endpoints](http://www.tutorialspoint.com/articles/spring-boot-actuator-a-production-grade-feature-in-spring-boot)
-   I love what they're doing at Stormpath, [and this latest screencast is a good example of why](https://stormpath.com/blog/microservices-spring-boot-screencast)
-   Did you know that Pivotal has classes you can take to [learn about Pivotal Cloud Foundry and Spring Cloud Services](http://pivotal.io/academy)?
-   the CAS project, series 5, continues [their march to GA and to a comprehensive Spring *Bootification* of the code in their latest release, M3](https://mmoayyed.github.io/2016/07/23/cas5-m3-release/)
