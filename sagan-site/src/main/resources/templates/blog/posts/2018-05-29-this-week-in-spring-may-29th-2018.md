# This Week in Spring - May 29th, 2018

_Engineering | Josh Long |  May 29, 2018 | 0 Comments_

Hi Spring fans and welcome to another installment of *This Week in Spring*! I finished last week in Barcelona, Spain, for the *epic* Spring IO 2018 event (were you there? why not? YOU SHOULD BE IT'S AMAZING!) and now I'm in San Francisco enjoying a restive 48 hours and soon I'll be off to sunny Saint Louis for the epic [Spring One Tour St. Louis event](https://springonetour.io/2018/st-louis) starting the 30th then it's off to splendid Singapore for [the Voxxed Singapore event](https://voxxeddays.com/singapore/). If you're in either region, [reach out](http://twitter.com/starbuxman)! I'd love to heard and/or see you!

Anyway, as usual, we've got a ton to cover so let's get to it!

-   Hi Spring fans! In last week's installment of *Spring Tips* we look at the Apache Camel enterprise integration framework and [its integration with Spring Boot](https://spring.io/blog/2018/05/23/spring-tips-apache-camel)
-   Don't miss Spring Cloud Stream lead [Oleg Zhurakousky's June 7 webinar on the latest and greatest in Spring Cloud Stream](https://content.pivotal.io/webinars/jun-7-spring-cloud-stream-whats-new-in-2-x-and-whats-next-webinar)
-   Do you really need any more reason to go to SpringOne Platform? Do *not* miss [this compelling invitation](https://spring.io/blog/2018/05/22/enjoy-quality-time-with-the-spring-and-reactor-communities-at-springone-platform) from Reactor project lead Stephane Maldini!
-   Check out the latest and greatest [support in the Cloud Foundry CLI](https://www.cloudfoundry.org/blog/cloud-foundry-cli-project-lead-updates/)
-   Check out the latest [in Cloud Foundry's container networking support](https://www.cloudfoundry.org/blog/new-updates-cloud-foundry-container-networking/)
-   Spring Data ninja and Redis-driver Lettuce maintainer Mark Paluch has a nice [post on using Redis Streams with Java](https://paluch.biz/blog/174-a-first-look-at-redis-streams-and-how-to-use-them-with-java.html)
-   This is an interesting post on [using Spring Cloud and Kubernetes and Jenkins X](https://salaboy.com/2018/05/24/spring-cloud-kubernetes-example/).
-   Oleksandr Slynko has a [inspiring post](https://medium.com/@slynko/kubernetes-with-cloud-foundry-2fef09efbdb6) on the hope of using Kubernetes with Cloud Foundry, based on Alexis Richardson's CNCF 20-20 Vision track talk
-   [Spring IO 2018 - Monitoring Spring Boot Applications with Spring Boot Admin // Speaker Deck](https://speakerdeck.com/joshiste/spring-io-2018-monitoring-spring-boot-applications-with-spring-boot-admin)
-   Assemberl, a small library to efficiently assemble results from querying, or merging of external data sources, or aggregation of microservices, [is now at 0.0.5](https://github.com/pellse/assembler). It offers a fluent DSL that uses Reactor. It was designed as a very lightweight solution to resolve the N + 1 queries problem when aggregating data, not only from database calls (e.g. Spring Data JPA, Hibernate) but from arbitrary data sources (relational databases, NoSQL, REST, local method calls, etc.)
-   Gerit Meier's slides on [Spring Data Neo4j, from Spring IO 2018, are now up](https://speakerdeck.com/meistermeier/time-to-graph-up-with-spring-data-neo4j). Neo4J is a graph-database that's amazing at all sorts of modeling.
-   The slides from Spring Data lead Oliver Gierke's talk on "REST design beyond the obvious", from Spring IO 2018, [are now available](https://speakerdeck.com/olivergierke/rest-beyond-the-obvious-api-design-for-ever-evolving-systems)
-   The slides from Mark Pakuch's Spring IO 2018 talk, ["Under the Hoods of Reactive Data Access"](https://speakerdeck.com/mp911de/under-the-hood-of-reactive-data-access-1), are now available.
-   The slides from Reactor ninja Simon Basle's Spring IO talk, ["Flight of the Flux," are now available](https://speakerdeck.com/simonbasle/flight-of-the-flux)
-   This is an oldie-but-a-goodie; it's Spring web-dude Rossen Stoyanchev's [comparison of Servlet and reactive web programming](https://twitter.com/infoq/status/1000142219154771969?s=12) in Spring Framework.
-   Ben Wilcock has a *fabulous* infographic [that looks at both Kubernetes and Pivotal Cloud Foundry](https://twitter.com/benbravo73/status/999959147406819333) and how and why they're both awesome.
