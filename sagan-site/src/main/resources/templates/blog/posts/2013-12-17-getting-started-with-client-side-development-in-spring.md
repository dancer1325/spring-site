# Getting Started with Client-Side Development in Spring

_Engineering | Craig Walls |  December 17, 2013 | 0 Comments_

Dear Spring Community,

At SpringOne, we launched a brand new Spring web site, including a collection of Getting Started Guides. These quick-hit guides are proving very popular as they get right to the point on how to use Spring to address several common tasks such as caching, messaging, and integration.

We are very pleased to announce the addition of several new Getting Started Guides that present Spring as providing services that back rich and mobile clients developed in Backbone, AngularJS, iOS, and other client-side frameworks and platforms. These guides include:

-   [Consuming a RESTful Web Service with AngularJS](https://spring.io/guides/gs/consuming-rest-angularjs/)
-   [Consuming a RESTful Web Service with jQuery](https://spring.io/guides/gs/consuming-rest-jquery/)
-   [Consuming a RESTful Web Service with rest.js](https://spring.io/guides/gs/consuming-rest-restjs/)
-   [Consuming a RESTful Web Service with Sencha Touch](https://spring.io/guides/gs/consuming-rest-sencha/)
-   [Consuming a RESTful Web Service with Backbone.js](https://spring.io/guides/gs/consuming-rest-backbone/)
-   [Consuming a RESTful Web Service from iOS](https://spring.io/guides/gs/consuming-rest-ios/)
-   [Consuming a RESTful Web Service with Spring for Android](https://spring.io/guides/gs/consuming-rest-android/)
-   [Building a RESTful Web Service with CORS](https://spring.io/guides/gs/rest-service-cors/)
-   [Messaging with STOMP over WebSocket](https://spring.io/guides/gs/messaging-stomp-websocket/)
-   [Using WebSocket and msgs.js to build an interactive web application](https://spring.io/guides/gs/messaging-stomp-msgsjs/)

We hope you'll find these guides to be a handy reference for backing your Javascript and mobile clients with Spring backends.
