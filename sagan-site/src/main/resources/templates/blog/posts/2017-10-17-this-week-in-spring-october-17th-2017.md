# This Week in Spring - October 17th, 2017

_Engineering | Josh Long |  October 17, 2017 | 1 Comment_

Hi Spring fans! Welcome to another installment of *This Week in Spring*! This week I've been in Dallas and Austin, Texas, first for customer meetings and then to attend and speak at [the epic Reactive Summit](https://www.reactivesummit.org/). If you're around, don't hesitate to [say hi on Twitter!](http://twitter.com/starbuxman) We've got a *lot* to cover so let's get to it!

Also, I'll be doing a webinar with our friends at OCI on November 9th, 2017, on [Grails for the Spring Developer, with my friend and Grails co-founder Jeff Scott Brown](https://objectcomputing.com/products/grails/webinar-grails-3-spring-developers/). I hope you'll join us!

-   Last week, SpringSource co-founder and Spring Framework lead [Juergen Hoeller won a JAX Special Jury Award for all of his considerable work since the very beginning of the Spring projects](https://spring.io/blog/2017/10/10/spring-co-founder-and-spring-framework-lead-juergen-hoeller-wins-jax-special-jury-award). Thanks to him, the special jury noted, "Spring Framework has never needed a complete rewrite in all of the years it has existed. Instead, it has continuously adapted to the changing trends." Thank you, Juergen, for all that you do.
-   Last week, I looked at how to use [Scala with a reactive Spring Boot and Spring Webflix-based application](https://spring.io/blog/2017/10/11/spring-tips-bootiful-reactive-scala) and how to interoperate with other Reactive Streams-compliant projects like Akka Streams.
-   Spring Data lead Oliver Gierke just announced [Spring Data Ingalls SR8](https://spring.io/blog/2017/10/11/spring-data-ingalls-sr8-released). This release includes a number of improvements and bug fixes and is a recommended upgrade for all those using the Ingalls release train.
-   Spring Boot [lead Phil Webb just announced Spring Boot 2.0.0.M5](https://spring.io/blog/2017/10/12/spring-boot-2-0-0-m5-available-now). The new release is [now available on the Spring Initializr](http://start.spring.io). It includes preliminary support for Java 9, OAuth 2 client support from Spring Security 5.0, JSON-B support, and a new Spring Session Actuator endpoint.
-   [Spring Tool Suite lead Martin Lippert just announced Spring Tool Suite 3.9.1](https://spring.io/blog/2017/10/12/spring-tool-suite-3-9-1-released). The new release updates to Eclipse Oxygen 1.a, includes JDK 9 and JUnit 5, updated the Spring Boot Dashboard to support insight of *running* beans in a Spring Boot release. This last thing is *so* cool! There's also a refactoring to convert `application.properties` to `application.yml`, and of course a slew of bug-fixes and improvements.
-   Spring ninja Stéphane Nicoll just announced [Spring Boot 1.5.8](https://spring.io/blog/2017/10/17/spring-boot-1-5-8-available-now) is now available. The new release includes over 40 fixes, improvements and dependency updates. This is a recommended update.
-   Spring Web Services lead Greg Turnquist [just announced Spring WS 2.4.1 and 3.0.0.RC1](https://spring.io/blog/2017/10/17/spring-web-services-2-4-1-release-3-0-0-rc1-released). The 2.4.1 has a few minor patches where 3.0.0.RC1 has a number of major upgrades. The new release baselines on Java 8 and Spring Framework 5.0. If you're building SOAP-based services, this is for you!
-   Have you seen the new [Spring Cloud integration for Google Cloud Platform](http://cloud.spring.io/spring-cloud-gcp/)? There's a *lot* to like here including integration with core GCP functionality like the project ID, and Runtime Config API, Google Cloud Pub/Sub, Google Cloud SQL, Google Cloud Storage, and Spring Cloud Sleuth integration with Google Stackdriver!
-   Spring Data lead [Oliver Gierke gave a really nice interview in this podcast](https://content.pivotal.io/podcasts/from-monolith-to-a-system-of-systems-ep-46?mkt_tok=eyJpIjoiTmpNMlpURmxPR0ZqWm1ZeCIsInQiOiJVWllnQzRDSG1XRFFqb3JaUzZSN0VHNnMyUEJpTzZXbXhOSlwvRVNESWRuOGJKdGhTTnlCSkVUdjRYaFo2ZWJcL3pRSGVKaUNoZWdGaHRsXC92WFIwRHZwZFJ5aFp1SUllQmFpVTVaa05KMCtKbEQ3UThnUDB5dTdJa3JcL3FKaVFQZzgifQ%253D%253D) on moving from a monolith to a system of systems. Definitely worth a listen!
-   Judy Wang has a nice look at [how Pivotal Cloud Foundry's "PCF Metrics" reduce mean-time-to-recovery](https://content.pivotal.io/blog/how-pcf-metrics-helps-you-reduce-mttr-for-spring-boot-apps-and-save-money-too?_lrsc=11ca73a3-c08a-4131-86af-9743179a2b51) with better insight..
-   Spring Framework founder and Atomist's CEO Rod Johnson [introduces Atomist's larger perspective of automation of development](https://the-composition.com/if-you-care-about-it-automate-it-with-atomist-20eccb8a28) in this interesting blog post. Atomist provide a number of tools that could be very useful for Spring developers.
-   our very own Kenny Bastani looks at using event driven architectures with a variety of Spring-based [technologies in this sample application](https://github.com/kbastani/event-stream-processing-microservices#packaging)
-   I loved Vedran Pavić's slides for his presentation, [*From Zero to Open-Sourcec Hero: Contributing to Spring Projects*](https://speakerdeck.com/vpavic/from-zero-to-open-source-hero-contributing-to-spring-projects-2)
-   Spring framework founder and Atomist CEO Rod Johnson expands on Atomist's perspective around the automation of development in this blog post. [This is rocket sauce for developers](https://the-composition.com/if-you-care-about-it-automate-it-with-atomist-20eccb8a28?source=userActivityShare-a17df5ec14a4-1508101154).
-   I did a webinar for the folks at JetBrains on Reactive Spring. That webinar [is now available here](https://blog.jetbrains.com/idea/2017/10/reactive-spring/?mail=event). The webinar introduces Reactor, Spring Framework 5.0 (and Spring WebFlux), Spring Data Kay, Spring Boot 2.0 (and the Actuator), Spring Security 5.0, and the `WebClient`, all in the Kotlin language! I hope you enjoy.
-   I thought this was pretty cool - Pivotal [has a TV commercial](https://www.ispot.tv/ad/A9Fp/dell-technologies-magic-with-ge-featuring-jeffrey-wright)!
-   Spring testing ninja Sam Brannen [put together a nice example](https://github.com/sbrannen/junit5-demo/blob/master/src/test/java/extensions/CaptureSystemOutput.java) on how to write a JUnit 5 Jupiter extension that replaces a common `@Rule` from the JUnit 4 programmming model.
-   Redmonk's James Governor has some insight into the death of Hudson, which Oracle publicly deprecated last week at Oracle OpenWorld, and the prevalence of technologies like Jenkins, [and rising stars like Pivotal's Concourse](https://redmonk.com/jgovernor/2017/10/12/oh-hai-jenkins-oh-bye-hudson-lets-keep-an-eye-on-concourse/).
-   Also, Redmonk's James Governor [also reviews some of the news from the Cloud Foundry Summit](https://redmonk.com/jgovernor/2017/10/11/some-thoughts-on-cloud-foundry-summit/) including the emerging details of Kubo, the Kubernetes implementation managed by Pivotal Cloud Foundry.
-   This is an oldie but a goodie from Tim Spann at Hortonworks [on creating a Spring Boot microservice to read Apache Phoenix Data](https://community.hortonworks.com/content/kbentry/56642/creating-a-spring-boot-java-8-microservice-to-read.html).
-   TechCrunch also had a roundup of the recent Cloud Foundry Summit news, focusing on [the integration of Kubernetes for containers into Cloud Foundry](https://techcrunch.com/2017/10/11/cloud-foundry-launches-its-online-marketplace-and-adds-native-kubernetes-support/amp/). This is ideal for stateful services like Elasticsearch.
-   JAXEnter's Jane Elizabeth has a nice look [at Spring Data Kay](https://jaxenter.com/spring-data-kay-137782.html), the recently released (reactive Spring Data implementation.
-   The JAX Innovation awards just took place and, in particular, we're elated to see that [Spring Framework lead Juergen Hoeller](https://jaxenter.com/winners-jax-innovation-awards-2017-137993.html) has won a JAX Special Jury Award. TL;DR: Spring is one of the largest and continuously maintained codebases and no other person in the Java community has done more to contribute to and shape such a codebase for so long. Thanks to Juergen, the codebase hasn't required a complete rewrite. As the announcement says, "Thank you, Jürgen, for everything you’ve done for Spring Framework!"
