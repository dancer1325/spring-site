# This Week in Spring - May 8th, 2017

_Engineering | Josh Long |  May 09, 2017 | 1 Comment_

Hi Spring fans! This week I'm in Austin, Texas for [OSCON](https://conferences.oreilly.com/oscon/oscon-tx/public/schedule/speaker/171564); London, UK, for [Devoxx UK](http://cfp.devoxx.co.uk/2017/talk/YDG-0684/Reactive_Spring); and Nice, France for [RivieraDev](http://rivieradev.fr/orateur/152)! If you're around, I'd love to talk to you so don't hesitate to say hi!

-   [Spring Framework 5 RC1 is now available!](https://spring.io/blog/2017/05/08/spring-framework-5-0-goes-rc1)
-   InfoQ has a nice post on the new [Spring Vault release](https://www.infoq.com/news/2017/05/spring-vault-ga)
-   Spring Integration lead Gary Russell [just announced Spring AMQP 2.0M4 is now available](https://spring.io/blog/2017/05/08/spring-amqp-2-0-0-milestone-4-is-available)
-   Spring Cloud ninja Ryan Baxter [just announced Spring Cloud Camden SR7 has been released](https://spring.io/blog/2017/05/08/spring-cloud-camden-sr7-released)
-   I liked this short-but-sweet look at the Pivotal Cloud Cache on Pivotal Cloud Foundry, [in terms of a Spring Boot application](http://theblasfrompas.blogspot.com/2017/04/spring-boot-application-for-pivotal.html)
-   Periodic reminder: [Spring Cloud Contract](https://cloud.spring.io/spring-cloud-contract/spring-cloud-contract.html) helps you solve the problem of integration testing distributed systems- don't sleep on this!
-   Spring ninja Stéphane Nicoll‏ [announced that there is now a starter for Reactive Redis support](https://twitter.com/snicoll/status/860149206186692611) on the Spring Initializr
-   I really enjoyed this discussion of the benefits of self-contained services (as compared to microservices) [on InfoQ](https://www.infoq.com/articles/scs-microservices-done-right)
-   Kamala Dasika has put together a nice look [going upstack when applying the 3 Rs of security](https://content.pivotal.io/blog/cloud-native-security-going-upstack-from-the-3-rs)
-   the [new Spring Cloud Data Flow page has a very convenient getting-started experience](http://cloud.spring.io/spring-cloud-dataflow/)
-   Praneeth Ramesh did a [nice job introducing Cloud Foundry for developers](https://dzone.com/articles/cloud-foundry-for-developers)
-   I liked this post by Michael Simons [digging into what makes a Spring Boot starter](http://info.michael-simons.eu/2017/04/30/digging-deep-into-spring-boots-starter/)
