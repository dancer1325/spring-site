# This Week in Spring - July 30, 2013

_Engineering | Josh Long |  July 31, 2013 | 0 Comments_

Welcome back to another installment of *This Week in Spring*! As usual, we've got a lot to cover so let's get to it. Don't forget that SpringOne 2GX 2013 early bird expires August 9th, [so hurry](http://www.springone2gx.com/conference/santa_clara/2013/09/register) to secure the discounted rate!

1.  Spring framework committer Rossen Stoyanchev has a great post on [Spring Framework 4.0 M2's support for WebSocket Messaging Architectures](http://blog.springsource.org/2013/07/24/spring-framework-4-0-m2-websocket-messaging-architectures/).
2.  Spring Shell lead Dr. Mark Pollack has announced that [Spring Shell 1.0.1.M1](http://www.springsource.org/node/22656) has just been released.
3.  [Spring Batch 2.2.1.RELEASE](http://www.springsource.org/node/22658) is now available. This release is mostly bug fixes and documentation improvements.
4.  I don't know if you've been following along, but we're starting to really [flesh out the SpringOne2GX 2013 schedule](https://springone2gx.com/conference/santa_clara/2013/09/schedule)! I'm looking forward to both seeing, and presenting, at many different talks this year. One talk I'd like to see is [Thymeleaf: improving your Spring view layer with natural templates](http://www.springone2gx.com/conference/santa_clara/2013/09/session?id=29453). I expect this year will be a *very* exciting year for a number of reasons, and I hope you'll share the experience with us.
5.  We've added some more SpringOne talks recently:  
    -   [Spring and Web Content Management](http://www.springone2gx.com/conference/santa_clara/2013/09/session?id=29824)
    -   [Spring Data Community Lightning Talks](http://www.springone2gx.com/conference/santa_clara/2013/09/session?id=29836)
    -   [Application Security Pitfalls](http://www.springone2gx.com/conference/santa_clara/2013/09/session?id=29837)
    -   [Hadoop - Just the Basics for Big Data Rookies](http://www.springone2gx.com/conference/santa_clara/2013/09/session?id=29835)
    -   [Spring Integration Internals](http://www.springone2gx.com/conference/santa_clara/2013/09/session?id=29823)
6.  Our pal Tobias Flohre has put together a [nice post comparing how the JSR 352 API compares to the Spring Batch](Tobias%20Flohre). Spring Batch 3.0 will be fully JSR 352 API compliant this fall by SpringOne, but was the [inspiration for the JSR in the first place](http://www.infoq.com/news/2013/06/ee7-spring-batch) -- Spring Batch 1.0 was released in 2008 and has been gathering steam ever since.
7.  Want to learn more about [Spring Scala](http://github.com/SpringSource/Spring-Scala)? Watch Spring Scala lead talk about [it at ScalaDays New York](http://parleys.com/play/51c1ad7ee4b0d38b54f4621e/chapter0/about).
8.  As I mentioned last week, you'd do well to also follow [*This Week in Cloud Foundry*](http://www.thisweekincf.com/blog/2013/07/28/this-week-in-cloud-foundry-july-28/), which has a lot of great content following last week's large [announcement of a partnership between Pivotal and IBM](http://blog.cloudfoundry.com/2013/07/24/cloud-foundry-is-open-and-pivotal-part-2/).  
    
9.  [The Reactor project](https://github.com/reactor/reactor) lead by [John Brisbin](http://twitter.com/jbrisbin) has just [announced](https://twitter.com/j_brisbin/status/362007966938959872) support for a [`@EnableReactor` annotation for Spring Java configuration](https://github.com/reactor/reactor-spring/blob/master/reactor-spring-context/src/test/groovy/reactor/spring/context/config/ReactorBeanDefinitionRegistrarSpec.groovy).
10.  ..Speaking of [Thymeleaf](http://www.thymeleaf.org/) (the open source, Spring MVC, HTML5 and Tiles-friendly view and templating engine), version 2.1 will have [parameterizable fragments](https://github.com/thymeleaf/thymeleaf/issues/181). Do you want to test them? Try the `2.1.0-SNAPSHOT` version when specifying your Maven repository-compatible coordinates.
11.  Our friend Johnathan Mark Smith is at it again, this time with [a video on using Spring Data MongoDB](https://www.youtube.com/watch?v=aY_JsfJM2eU). Definitely worth a look.
12.  Check out a webinar next month *[taming coupling & cohesive problems with modularity and Spring](https://www.springsource.org/node/22645)* with Param Rengaiah.
