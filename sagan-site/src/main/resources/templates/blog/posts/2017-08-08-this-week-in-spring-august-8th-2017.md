# This Week in Spring - August 8th, 2017

_Engineering | Josh Long |  August 08, 2017 | 0 Comments_

Hi Spring fans and welcome to another installment of *This Week in Spring*! This week I'm presenting with my [buddy Matt Raible](http://twitter.com/mraible) on the VJUG (Virtual JUG) on Progressive Web Applications with Spring Boot and Angular 4, this Wednesday. [Join us](https://www.meetup.com/virtualJUG/events/241270566/)! Then it's off to New York City, NY where I'll be working with my buddy Marcin Grzejszczak on an exciting new project.

-   Spring IO Platform lead Andy Wilkinson [just announced Spring IO Platform Brussels-SR4](https://spring.io/blog/2017/08/04/spring-io-platform-brussels-sr4). This release revises the versions for Spring Batch 3.0.8, Spring Boot 1.5.6, Spring Data Ingalls SR6, Spring Framework 4.3.10, Spring Integration 4.3.11, and Spring Retry 1.2.1.
-   The [first milestone of Spring Cloud Data Flow 1.3.0](https://spring.io/blog/2017/08/07/spring-cloud-data-flow-1-3-0-m1-released) is out! Lots of nice new features including a flashy new user interface based on Angular 4, better documentation, better test coverage, and so much more.
-   Spring Cloud Pipelines lead Marcin Grzejszczak just announced [Spring Cloud Pipelines 1.0.0.M6](https://spring.io/blog/2017/08/04/spring-cloud-pipelines-1-0-0-m6-released). This release supports a powerful mechanism called *pipeline descriptors* to simplify pipeline definitions.
-   [Spring Cloud Data Flow for Kubernetes v1.2.2.RELEASE](https://github.com/spring-cloud/spring-cloud-dataflow-server-kubernetes/releases/tag/v1.2.2.RELEASE) is available now. This release improves the documentation, updated server deployment scripts aligned with Helm Chart, and so much more.
-   As stewards of the RabbitMQ project, [we're happy to offer this course introducing RabbitMQ](https://academy.pivotal.io/confirm-course?courseid=EDU-1099).
-   [Ernest Ortuño introduces Trampoline](https://dzone.com/articles/trampoline-start-and-stop-spring-boot-based-servic), an open-source community project that makes starting and stopping Spring Boot-based services on your local machine a lot easier.
-   I liked this post from RedMonk analyst James Governor [that looks at the position that Dell](https://redmonk.com/jgovernor/2017/08/07/so-about-this-dell-digital-transformation-story-pivots/) (parent of Pivotal) holds in the market.
-   I was asked for insight on versioning APIs and I did the *right thing* - I asked Spring Data lead Oliver Gierke who then [referenced this epic post, *Roy Fielding on Versioning, Hypermedia, and REST*](https://www.infoq.com/articles/roy-fielding-on-versioning).
-   InfoQ has a nice post on the Spring Cloud Function framework, [which supports the functions-as-a-service development paradigm](https://www.infoq.com/news/2017/08/Spring-Cloud-Function-Framework). This is a good read if you haven't gotten to know this exciting new project.
-   Interested in the proposed evolution of Java? Check out the [videos from the JVM Language Summit](https://www.youtube.com/playlist?list=PLX8CzqL3ArzXJ2EGftrmz4SzS6NRr6p2n).
-   My talk, [*Reactive Spring*](https://www.youtube.com/watch?v=EDB24onZxsA&list=PLRsbF2sD7JVq8QYW0vlbOS2JuXUWaWMnT&index=6), from Devoxx UK, is now available online. Check it out!
-   This [Chinese-language post](http://mp.weixin.qq.com/s/n3iNeH9lWxspqd4cFTO-_g) (called "Spring Boot、Dubbo项目Mock测试踩坑与总结") looks at mocking objects in a Spring Boot-based version of the Alibaba Dubbo service framework. (I think)
-   I loved [this InfoWorld article on Atomist](http://www.infoworld.com/article/3213386/application-development/chatops-for-the-modern-developer-courtesy-of-springs-founder.amp.html), the developer lifecycle-management solution that integrates chatops. Atomist was founded by Spring creator Rod Johnson and, so, integrates amazingly well with Spring.
-   NewRelic talk about why they [chose Spring Boot for their New Relic Alerts product](https://blog.newrelic.com/2017/05/18/alerts-microservices-environment-spring-boot/) after comparisons with Dropwizard, Restlet, Vert.x, and Spark.
-   Have you seen Spring Boot and Spring Cloud co-founder [Dr. Dave Syer's Spring Boot thin-launcher plugin](https://github.com/dsyer/spring-boot-thin-launcher)? This supports a thin-jar layout - where the library dependencies are resolved on launch - for your Spring Boot application.
-   Andrey Cheptsov talks about a [very cool new feature in IntelliJ IDEA 2017.2.1](https://blog.jetbrains.com/idea/2017/08/intellij-idea-2017-2-1-junit-5-rc2-spring-run-dashboard-and-its-faster/), a Spring Run Dashboard, that lets you manage and launch multiple run configurations at once. This is so cool! If you're using Spring Tool Suite, of course, then you've got a similar experience in the Spring Boot Dashboard.
-   Spring Cloud Pipelines and Spring Cloud Contract lead Marcin Grzejszczak and I did a talk looking [at consumer driven contracts with Spring Cloud Contract a Devoxx Poland](https://www.youtube.com/watch?v=IiK9A9nQ6NU). That video is online and well-worth a watch!
