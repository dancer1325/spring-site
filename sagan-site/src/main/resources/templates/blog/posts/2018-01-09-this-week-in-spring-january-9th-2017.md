# This Week in Spring - January 9th, 2017

_Engineering | Josh Long |  January 09, 2018 | 3 Comments_

Hi Spring fans! Welcome to another installment of *This Week in Spring*! This week I'm off to Germany where I'll be speaking at the [Java User Group in Münster on Wednesday night](http://www.jug-muenster.de/cloud-native-java-mit-spring-boot-10-1-2018-1489/). Then, it's off to Solingen for a [Cloud Native day on the 12th (this Friday) where I'll be presenting all afternoon](https://www.codecentric.de/2017/11/30/cloud-native-java-day-spring-rockstar-josh-long/) - register now! And, if you're closer to the Pacific ocean than the Atlantic ocean, join me next Monday in Hawaii and we'll talk about [all things Spring at the very promising LavaOne conference](http://events.eventzilla.net/e/lavaone-2018-2138921167).

As usual, we've got a lot to cover so let's get to it.

-   There's [new support for auto-configuring the Reactor-based Cloud Foundry Java client in Spring Cloud](https://github.com/spring-cloud/spring-cloud-cloudfoundry/tree/master/spring-cloud-cloudfoundry-commons). This is exciting for me because that Java client is *such* a powerful tool that takes *so* long to configure.
-   There's a new guide on [using Spring Cloud contract](https://spring.io/guides/gs/contract-rest/) by community legend Erdem Günay.
-   The Baeldung blog have a nice look at [bootstrapping Hibernate 5 with Spring](http://www.baeldung.com/hibernate-5-spring)
-   The first raft of [Spring Days events](http://springdays.io/?_lrsc=d6800d25-1146-4c09-95f8-28afef9f5d71) have been announced! Do *not* miss these events in a city near you! I'll be there along with other Pivotal peeps. Join us.
-   The [Apigee Edge Service Broker 3.0 for Pivotal Cloud Foundry](https://apigee.com/about/blog/api-technology/apigee-edge-service-broker-300-pivotal-cf-here) has just arrived.
-   Check out this Spanish-language [post on using Spring Cloud Contract](http://qajungle.com/consumer-driven-contract-testing-con-spring-cloud-contract/).
-   The [Resilience4J project is an interesting looking library](https://github.com/resilience4j/resilience4j) containing implementations of things like circuit breakers and retry handlers and it has nice looking Spring Boot integrations. I hope Spring Cloud one day integrates this. In the meantime, check it out.
-   Epic Spring Data for Apache Geode lead [John Blum gave an interview to the epic Software Engineering Daily podcast](https://softwareengineeringdaily.com/2018/01/04/spring-data-with-john-blum/): give it a listen.
-   Dan Vega has an interesting video [on installing the Spring Boot CLI in a Windows environment](https://www.youtube.com/watch?v=aPrlbE8E_bU&app=desktop)
-   I [gave an interview to Faros Belgium](https://www.youtube.com/watch?v=9Ef2Y4Dtp1I) during last month's epic SpringOne Platform 2017 event.
-   The [Cloud Foundry Foundation security team has detailed the response for Cloud Foundry users for the Spectrum and Meltdown flaws](https://cloudfoundry.org/meltdown-spectre-attacks/) recently discovered in CPUs. These kinds of vulnerabilities are *exactly* where a centralized platform like Cloud Foundry can make all the difference in the world - mitigate the issue centrally and see it deployed automatically.
-   Tristan Perry put together [a nice Spring Cloud microservices tutorial](https://www.tristanperry.com/java8/spring/2017/12/23/java8-spring-cloud-microservice-demo.html).
