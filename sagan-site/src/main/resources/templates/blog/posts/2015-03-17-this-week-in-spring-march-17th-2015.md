# This Week in Spring - March 17th, 2015

_Engineering | Josh Long |  March 17, 2015 | 0 Comments_

Welcome to another installment of *This Week in Spring*! This week I've been busily preparin for a workshop [I'll be giving at QCon São Paulo next week](http://qconsp.com/speaker/josh-long). If you're at the show, I hope you'll come out to say hi, ask questions and we'll talk Spring and Cloud Foundry. **Obrigado**!

1.  The replay on my humble webinar [introducing Spring Session is online, check it out](https://spring.io/blog/2015/03/12/webinar-replay-introducing-spring-session)!
2.  Janne Valkealahti has just announced the latest release [of Spring Security Kerberos 1.0.0.RC1](https://spring.io/blog/2015/03/11/spring-security-kerberos-1-0-0-rc1-released), packed with new features and now - at long last!! - near GA! Go Janne and team, go!
3.  Spring and Groovy & Grails Tool Suite lead [Martin Lippert just announced STS and GGTS 3.6.4](https://spring.io/blog/2015/03/11/spring-tool-suite-and-groovy-grails-tool-suite-3-6-4-released).
4.  I put together a post [on getting started with Spring Data Couchbase](https://spring.io/blog/2015/03/16/getting-started-with-couchbase-and-spring-data-couchbase)
5.  Spring XD ninjas Marius Bogoevici and Dr. Mark Pollack are going to be giving a webinar on April 28th that [introduces using Spring XD and Apache Kafka](https://spring.io/blog/2015/03/17/webinar-reactive-data-pipelines-with-spring-xd-and-kafka) to create reactive data-pipelines.
6.  I'll be giving a webinar on April 21st that introduces [how to make smart service-to-service calls with Spring Cloud](https://spring.io/blog/2015/03/17/webinar-smarter-service-to-service-invocations-with-spring-cloud).
7.  This Week's SpringOne2GX 2014 replays are out! Check out [Asychronous design with Spring and RTI: 1M events per second](http://spring.io/blog/2015/03/17/springone2gx-2014-replay-asychronous-design-with-spring-and-rti-1m-events-per-second), [Building a Recommendation Engine with Spring and Hadoop](http://spring.io/blog/2015/03/17/springone2gx-2014-replay-building-a-recommendation-engine-with-spring-and-hadoop), and author Peter Harrington's [Become a data-driven Organization with Machine Learning](http://spring.io/blog/2015/03/17/springone2gx-2014-replay-become-a-data-driven-organization-with-machine-learning).
8.  Curious about the why Pivotal considers iteself at the nexus of [Apps, Data and Cloud](http://spring.io/blog/2015/03/17/springone2gx-2014-replay-apps-data-cloud-what-does-it-all-mean)? Check out Matt Stine's SpringOne2GX replay for the answer.
9.  the Vaadin web framework now has [**official** support for Spring and Spring Boot](https://vaadin.com/wiki/-/wiki/Main/Vaadin+Spring). This support is the official stamping of the [code that Peter Holmst and I started](https://github.com/peholmst/vaadin4spring), so I couldn't be more excited to see how far it's come and to see it finally graduate to an officially supported module.
10.  Intellij 14.1 is packed with all sorts of amazing Spring Boot goodness! Check out this post from [JetBrain's Yann Cébron on some of the amazing new features](http://blog.jetbrains.com/idea/2015/03/develop-spring-boot-applications-more-productively-with-intellij-idea-14-1/) and be sure to register for this March webinar in March with our [own Stéphane Nicoll and Yann](http://info.jetbrains.com/IDEA-Webinar-March2015-registration.html) covering some of these amazing features!
11.  Heroku have put together a nice writeup [on using Spring Cloud to build cloud native applications on Heroku](https://blog.heroku.com/archives/2015/3/3/managing_your_microservices_on_heroku_with_netflix_s_eureka)
12.  Check out this video of [Spring framework lead Juergen Hoeller recorded from Voxxed Days Austria](http://www.infoq.com/interviews/spring-today-past-and-future)
13.  Alex Williams put together a nice post [on using Apache Brooklyn - an application blueprint and management system which supports a wide range of software and services in the cloud - to automate Cloud Foundry with their custom service broker](http://thenewstack.io/cloud-foundry-and-apache-brooklyn-for-automating-paas-with-a-service-broker/). The idea is simple: a Cloud Foundry service broker talks to backend services (it's what drives the flow from `cf services` and `cf create-service`). Apache Brooklyn exposed as a service broker in this way is powerful; you only need one broker to expose the catalog of Brooklyn blueprints.
14.  Our pal Matt Stine just published a free e-book for O'Reilly on [building cloud-native architectures](http://blog.pivotal.io/cloud-foundry-pivotal/news-2/new-book-explains-how-cloud-native-application-architectures-drive-the-innovation-cycle).
15.  There are a few case studies on the Pivotal blog today. First, Jarrod Vawdrey's post on using data-science to [predict TV viewer behavior and formulate a hit TV show is awesome](http://blog.pivotal.io/data-science-pivotal/case-studies-2/using-data-science-to-predict-tv-viewer-behavior-and-formulate-a-hit-tv-show)
16.  Dwayne Forde has put together a nice case study on [How Pivotal Performs Continuous Delivery of Mobile Services](http://blog.pivotal.io/cloud-foundry-pivotal/case-studies-2/how-pivotal-performs-continuous-delivery-of-mobile-services).
