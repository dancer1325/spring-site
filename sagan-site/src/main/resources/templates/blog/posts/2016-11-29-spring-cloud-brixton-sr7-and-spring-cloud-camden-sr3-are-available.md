# Spring Cloud Brixton.SR7 and Spring Cloud Camden.SR3 Are Available

_Releases | Ryan Baxter |  November 29, 2016 | 0 Comments_

On behalf of the team, I am pleased to announce that Brixton Service Release 7 and Camden Service Release 3 of [Spring Cloud](http://cloud.spring.io) are available today. The releases can be found in our [Spring Release repository](http://repo.spring.io/release) and [Maven Central](http://search.maven.org/#search%7Cga%7C1%7Ca%3A%22spring-cloud-dependencies%22).

These releases include primarily bug fixes.

#### [](#the-following-modules-were-updated-as-part-of-brixtonsr7)The following modules were updated as part of Brixton.SR7:

-   Spring Cloud Config [1.1.3.RELEASE](https://github.com/spring-cloud/spring-cloud-config/milestone/21?closed=1)
-   Spring Cloud Netflix [1.1.7.RELEASE](https://github.com/spring-cloud/spring-cloud-netflix/milestone/30?closed=1)
-   Spring Cloud Sleuth [1.0.11.RELEASE](https://github.com/spring-cloud/spring-cloud-sleuth/milestone/17?closed=1)

#### [](#the-following-modules-were-updated-as-part-of-camdensr3)The following modules were updated as part of Camden.SR3:

-   Spring Cloud Commons [1.1.6.RELEASE](https://github.com/spring-cloud/spring-cloud-commons/milestone/20?closed=1)
-   Spring Cloud Config [1.2.2.RELEASE](https://github.com/spring-cloud/spring-cloud-config/milestone/26?closed=1)
-   Spring Cloud Netflix [1.2.3.RELEASE](https://github.com/spring-cloud/spring-cloud-netflix/milestone/33?closed=1)
-   Spring Cloud Consul [1.1.2.RELEASE](https://github.com/spring-cloud/spring-cloud-consul/milestone/13?closed=1)
-   Spring Cloud Sleuth [1.1.0.RELEASE](https://github.com/spring-cloud/spring-cloud-sleuth/milestone/17?closed=1)
-   Spring Cloud Contract [1.0.2.RELEASE](https://github.com/spring-cloud/spring-cloud-contract/milestone/7?closed=1)

And, as always, we welcome feedback: either on [GitHub](https://github.com/spring-cloud), on [Gitter](https://gitter.im/spring-cloud/spring-cloud), on [Stack Overflow](http://stackoverflow.com/questions/tagged/spring-cloud), or on [Twitter](https://twitter.com/SpringCloud).
