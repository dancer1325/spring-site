# Groovy 2.1 released

_Engineering | Guillaume Laforge |  January 25, 2013 | 0 Comments_

The Groovy team is pleased to announce the **release of Groovy 2.1.0**.

With over **1.7 million downloads in 2012**, a **strong ecosystem** of successful projects like [Grails](http://grails.org/ "Grails web application stack"), [Gradle](http://www.gradle.org/ "Gradle build automation system"), [Spock](http://code.google.com/p/spock/ "Spock testing framework") or [Griffon](http://griffon.codehaus.org/ "Griffon rich desktop application framework") built on Groovy, the **Groovy programming language** continues its development and adoption, refines existing features and evolves new ones.

In this new release, Groovy 2.1:

-   **offers full support for the JDK 7 “invoke dynamic”** bytecode instruction and API,
-   goes beyond conventional static type checking capabilities with a **special annotation to assist with documentation and type safety of Domain-Specific Languages** and adds **static type checker extensions**,
-   provides **additional compilation customization** options,
-   features a **meta-annotation facility** for combining annotations elegantly,
-   and provides various other enhancements and minor improvements.

Please read all the details about the **new features and improvements** in the **[Groovy 2.1 release notes](http://bit.ly/groovy21notes "Groovy 2.1 release notes")** document.

You can **download Groovy 2.1.0** from the [Download area](http://groovy.codehaus.org/Download?nc "Download Groovy 2.1"), and have a look at the [JIRA tickets](http://bit.ly/groovy21jira "Groovy 2.1 JIRA release notes") we worked on.

The Groovy team is looking forward to your feedback on this new release!
