# This Week in Spring - June 28, 2016

_Engineering | Josh Long |  June 28, 2016 | 2 Comments_

Welcome to another installment of *This Week in Spring*! This week I'm in Munich and Wolfsburg, Germany, visiting with customers, then it's off to Seoul, South Korea, for customers and some user group talks.

![](https://raw.githubusercontent.com/joshlong/meh/master/wolfsburg-28-06-2016.jpg)

This week we've got a *lot* of exciting Spring Cloud Data Flow news, coincident with the new Spring Cloud Data Flow RC1

-   The good Dr. Pollack, co-lead of the Spring Cloud Data Flow project, [just announced Spring Cloud Data Flow RC1](http://spring.io/blog/2016/06/21/spring-cloud-data-flow-1-0-rc1-released) which has updated deployers supporting deployments [locally](http://docs.spring.io/spring-cloud-dataflow/docs/1.0.0.RC1/reference/htmlsingle/#getting-started-deploying-spring-cloud-dataflow) and to [Apache YARN](https://spring.io/blog/2016/06/21/spring-cloud-data-flow-for-apache-yarn-1-0-0-rc1-released), [Kubernetes](https://spring.io/blog/2016/06/21/spring-cloud-data-flow-for-kubernetes-1-0-0-rc1-released), [Mesos](https://spring.io/blog/2016/06/21/spring-cloud-data-flow-for-apache-mesos-1-0-0-rc1-released), and of course [Cloud Foundry](https://spring.io/blog/2016/06/21/spring-cloud-data-flow-for-cloud-foundry-1-0-0-m3-released), an incredible integration of Spring Cloud Flo (a visualization for Spring Cloud Data Flow streams) into the Spring Cloud Data Flow Dashboard, improved OAuth support and *so* much more!
-   I *really* enjoyed Mark Paluch's *excellent* blog pulling back the veil of obscurity around the security secrets sensation sweeping the nation, Hashicorp Vault, and [how Spring Cloud Vault works](http://spring.io/blog/2016/06/24/managing-secrets-with-vault).
-   I loved this discussion between our very [own Phil Webb, Spring Boot co-founder, and Andrew Clay Shafer, one of our cloud luminaries](https://www.youtube.com/watch?v=cgTvvdxyIZs)!
-   Have you seen [our new Spring Cloud Task Starters?](http://cloud.spring.io/spring-cloud-task-app-starters/) From the page: "the Spring Cloud Task Application Starters are standalone executable applications that can be used for on-demand use-cases such as database migration, machine learning, and scheduled operations. These applications can run independently on variety of runtime platforms including: Cloud Foundry, Apache Yarn, Apache Mesos, Kubernetes, Docker, or even on your laptop."
-   our friends at JDriven put together a nice post on [creating fully executable `.jar`s with the Spring Boot Maven plugin](http://blog.jdriven.com/2016/06/spicy-spring-creating-fully-executable-jar/)
-   Jakub Narloch put together a great post on a custom auto-configuration to run work [on various RxJava schedulers with his custom component model based on Spring Boot](https://jmnarloch.wordpress.com/2016/06/22/spring-boot-rxjava-scheduler/)
-   Stormpath's own Micah Silverman put together a great post [on connecting a Spring Boot application with Stormpath Authentication in 15 minutes](https://www.javacodegeeks.com/2016/06/simple-web-app-spring-boot-spring-security-stormpath-15-minutes.html)
-   Trevor Page put together a [fun piece](http://coderscampus.com/ep01-spring-boot-rocks/) on how the Spring team seem hell-bent on making our lives as Java programmers easier and easier." I get the feeling we're making fans.. :-)
-   I haven't read this yet, but this [book on Spring Cloud Data Flow looks interesting](http://www.springer.com/kr/book/9781484212394)!
-   In New York City? Check out this meetup [on Spring Cloud Data Flow](http://www.meetup.com/NYC-SUG/events/232197164/) with Spring Cloud Data Flow lead Mark Pollack himself!
-   our very own Michael Coté put together a nice post, ["Multi-Cloud is a Safety Belt for the Speed-Freaks" for InfoQ](https://www.infoq.com/articles/multi-cloud-choice). His posts (and his talks and podcasts!) are insightful and often hilarious.
