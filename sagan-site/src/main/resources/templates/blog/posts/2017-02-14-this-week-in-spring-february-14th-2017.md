# This Week in Spring - February 14th, 2017

_Engineering | Josh Long |  February 14, 2017 | 0 Comments_

Welcome to another installment of *This Week in Spring!* It's Valentines Day for some, and so happy Valentines day to you! This week I'm in Memphis, Tennessee for business and then it's off to Saint Louis where I'll be presenting on [Reactive Spring with my buddy Mark Heckler (join us!)](https://www.meetup.com/Saint-Louis-Spring-Meetup/events/236875279/), then it's off to Atlanta, Georgia, ahead of next week's [big DevNexus show](https://www.devnexus.com/s/speakers/1284). Check out [what Pivotal is up to at the event!](https://spring.io/blog/2017/02/01/spring-team-at-devnexus-2017)

As usual, we've got a lot to cover so let's get going!

-   Last week, in another installment of Spring Tips, I [looked at distributed tracing with Spring Cloud Sleuth and Zipkin](https://spring.io/blog/2017/02/08/spring-tips-distributed-tracing-with-zipkin)
-   Spring Cloud Task lead Michael Minella has just announced [that Spring Cloud Task 1.2.0.M1](https://spring.io/blog/2017/02/09/spring-cloud-task-1-2-0-m1-is-now-available)
-   Marcin Grzejszczak has just announced [Spring Cloud Pipelines 1.0.0.M3 with complete support for blue-green deployment on Cloud Foundry](https://spring.io/blog/2017/02/09/spring-cloud-pipelines-1-0-0-m3-released)
-   check out [this SpringOne platform 2016 talk on Spring Data Hazelcast repositories](https://spring.io/blog/2017/02/13/springone-platform-2016-replay-spring-data-hazelcast-fluently-accessing-distributed-repositories)
-   there's a good article on [the Spring Boot Actuator support on Pivotal Web Services](https://dzone.com/articles/spring-boot-actuator-a-complete-guide)
-   this S1P talk on streaming live [data in the Hadoop ecosystem looks interesting](https://spring.io/blog/2017/02/13/springone-platform-2016-replay-streaming-live-data-and-the-hadoop-ecosystem)
-   Oleg Zhurakousky gives a nice look at [using Apache NiFi with Spring](https://spring.io/blog/2017/02/13/springone-platform-2016-replay-spring-with-apachenifi) with this S1P talk
-   our pal Andreas Evers gives a nice look at [writing comprehensive documentation with Spring REST Docs in this S1P talk](https://spring.io/blog/2017/02/13/springone-platform-2016-replay-writing-comprehensive-and-guaranteed-up-to-date-rest-api-documentation)
-   RedHat's Christian Posta's S1P talk on [real-world microservices with Spring Cloud and Kubernetes looks interesting](https://spring.io/blog/2017/02/13/springone-platform-2016-replay-real-world-microservices-with-spring-cloud-netflix-oss-and-kubernetes)
-   Want to learn more on Spring Cloud Data Flow? Check out [this S1P talk, *Spring Cloud Data Flow: Orchestrate All The Things*](https://spring.io/blog/2017/02/13/springone-platform-2016-replay-orchestrate-all-the-things-with-spring-cloud-data-flow)
-   Target have just open-sourced [Winnaker](http://target.github.io/infrastructure/Win_the_cloud_with_Winnaker) their auditing tool for Spinnaker. Spinnaker is Netflix's Spring Boot-powered continuous delivery platform. Pivotal, Netflix, Google and many others besides are all contributors to Spinnaker. You can run Spinnaker [on Cloud Foundry, too](https://cloud.spring.io/spring-cloud-spinnaker/)
-   I thought this interview with [Michael Minella on short lived microservice tasks was a great listen](https://soundcloud.com/pivotal-insights/put-an-end-to-it-managing-finite-microservices-tasks-ep-17)
-   hashicorp have just [unveiled their Vault Cloud Foundry service broker](https://www.hashicorp.com/blog/cloud-foundry-hashicorp-vault-service-broker.html)
-   check out 451Research's look [at Spring Cloud Data Flow](https://451research.com/report-short?alertid=305&contactid=0036000001lAfK6AAK&entityId=91591&type=mis&utm_campaign=market-insight&utm_content=newsletter&utm_medium=email&utm_source=sendgrid&utm_term=91591-Pivotal+pitches+Spring+Cloud+Data+Flow+for+continuous+data+processing)
-   This talk looks at how [Instagram uses RabbitMQ to handle asynchronous tasks at scale](https://www.youtube.com/watch?v=E708csv4XgY)
-   Chris Richardson has updated his [microservices pattern portal to include new patterns](http://microservices.io/patterns/index.html) - *service deployment platform*, *decomposition by subdomain*, *decomposition by business capability* and *externalized configuration*.
-   Randy Shoup provides a [fascinating look at autonomous teams in this interview](https://twitter.com/randyshoup/status/831215732234989568)
-   want to [use Java Server Faces and Spring Boot](https://github.com/joinfaces/joinfaces)?
-   the Baeldung blog has a nice [look at how to create executable `.jar` applications](http://www.baeldung.com/executable-jar-with-maven). It looks at various options, including the Spring Boot fat-`.jar` approach.
-   our pal Nicolas Frankel is back at it again, this time [with a look at how to achieve a self-contained `.jar` that is *also* signed](https://blog.frankel.ch/jvm-security/2/#gsc.tab=0).
